## 2. 顧問接觸與診斷

### 2.1 10 題快速問卷結果

均分 **0.8 → L1**。工具使用 Q1=1（公務員私下使用受限），治理 Q6=1、下一步 Q8=1（缺 AI Champion）。

### 2.2 25 題六構面

| 構面 | 均分 |
| --- | ---: |
| 工具使用 | 1.2 |
| 知識沉澱 | 1.5（公文歸檔文化） |
| 流程標準化 | 2.5（法規流程嚴謹） |
| 系統整合 | 1.0 |
| Agent 應用 | 0.2 |
| 治理 ROI | 1.3 |

**洞察**：流程標準化最高（法規驅動），但工具與整合低。公部門導入特性 — 採購慢、跨局處協調難，但一旦標竿建立可全市推廣。

## 3. 課程比例建議

| Level | 比例 | 重點 |
| ---: | ---: | --- |
| L1 | 45% | 先導 3 局處全員；地端部署；公文 / 市民資料紅線；公務員 AI 使用規範 |
| L2 | 25% | 公文 / 1999 / 採購 Skill |
| L3 | 22% | 公文摘要、市民 FAQ、跨局處知識問答 Workflow |
| L4 | 8% | 市民服務 Agent（先導，1999 場景） |
| L5 | 0% | 不導入 |

理由：公部門治理門檻最高（個資 + 公開法 + 資安法），L1 須 45%。採購週期長，地端基礎建設需提前規劃。

## 4. 課中產出資產

### L1 OpenWebUI (12 週，分局處)

- 800 帳號（先導 3 局處：民政、社會、研考）
- **全地端部署**：政府機房 + Llama 70B + 地端 Embedding
- 公務員 AI 使用規範（對標個資法 + 政府資訊公開法）
- 模型可見性：依局處與職等分層
- Prompt Library 38 條（皆經政風 + 法制預審）
- Audit Log：符合資安管理法稽核要求

### L2 Skill AI (10 週)

10 個 Skill：

| # | Skill | Owner |
| --- | --- | --- |
| 1 | 公文摘要與分文建議 | 文書科 |
| 2 | 公文初稿草擬（制式類） | 各科室 |
| 3 | 市民 1999 案件分類 | 1999 中心 |
| 4 | 市民陳情回覆草稿 | 各局處 |
| 5 | 法規條文白話化說明 | 法制 |
| 6 | 採購規格書草擬輔助 | 採購 |
| 7 | 跨局處知識問答（RAG） | 研考 |
| 8 | 會議紀錄與決議追蹤 | 秘書 |
| 9 | 新進公務員 onboarding Q&A | 人事 |
| 10 | 統計報表 narrative 草擬 | 主計 |

### L3 n8n Workflow (10 週)

4 個 Workflow PoC（皆地端）：

1. **公文摘要 Pipeline** — 公文系統 → Skill #1 → 承辦人 review queue
2. **1999 案件分流** — 1999 系統 → Skill #3 → 分派各局處 + 優先級
3. **跨局處知識問答** — 員工問 → RAG over SharePoint + 公開資料 → 答 + 來源
4. **市民陳情回覆輔助** — 陳情案 → Skill #4 → 承辦人 review

### L4 Hermes Agent (6 週)

**市民服務 Agent（1999 先導）**：
- Wiki：市政 FAQ、各局處業務、常見陳情類型、處理 SOP
- Skills：#3, #4
- 任務卡：1999 案件進線 → Agent 分類 + 草擬初步回覆 + 標示需人工的部分
- **Reviewer：1999 中心主管簽核**
- Evidence：來源 100% 追溯
- Gate 4A-4E + 額外 Gate 4G（政風 + 法制簽署）

### L5 ClawTeam

**不導入**。

## 5. 八階段顧問分析

### Stage 1 現況掌握
- 訪談：數位局長、研考會主委、3 局處長、文書科、1999 中心、政風、法制
- 痛點密度：公文（22 局處 100%）、1999 重複問答（1999 中心 100%）、跨局處知識（研考 90%）

### Stage 2 願景對齊
- 數位局長願景：24 個月內公文處理時間 -40%、1999 一線解決率 +30%、建立可全市複製的標竿
- Sponsor = 數位局長 + 研考會主委

### Stage 3 產業標竿
- 國際：新加坡 GovTech、愛沙尼亞 e-Estonia（願景參考）；台灣：數位部相關 PoC
- 採全地端，對標個資法 + 資安管理法

### Stage 4 差距分析

Missing/Broken/Redundant：
- **Missing**：AI 治理規範、地端基礎建設、AI Champion 編制、Skill Library
- **Broken**：公文逐字閱讀（每件 15-30 min）；1999 重複問答（40% 案件可標準化）
- **Redundant**：22 局處各自的知識散落（無統一檢索）

Impact × Effort：
- Quick Win：跨局處知識問答（L3-W3）；1999 分流（L3-W2）
- Big Bet：公文摘要 Pipeline（L3-W1，需串公文系統）
- Avoid：市民直接面對的開放式 AI chatbot（資訊正確性風險）

### Stage 5 高階問題定義

```
CONTEXT: 數位部推動公部門 AI 應用；市民對市府服務速度期待提高；
         公文量年增 8%，人力凍結。
TENSION: 法規流程嚴謹 (L2.5) 但無 AI 工具；跨局處知識不流通。
COI: 不行動 24 個月：(1) 公文積壓惡化 (2) 1999 滿意度下滑 (3) 落後其他縣市標竿。
DESIRED: 24 個月先導 3 局處達 L3-L4；公文處理 -40%；1999 一線解決 +30%；產出可全市複製模式。
CONSTRAINT: 預算分年編列；全地端；採購週期長；每應用須政風 + 法制 + 資安三審；分局處導入。
```

### Stage 6 路徑圖

| Phase | 月 | 主要交付 | Gate | KPI |
| --- | --- | --- | --- | --- |
| 1 | 1-6 | 採購 + 地端基建 + AI 治理規範 | 前置 Gate | 機房就緒、規範三審通過 |
| 2 | 7-12 | L1 先導 3 局處 800 人 | Gate 1 | 員工週使用 ≥ 60% |
| 3 | 13-18 | L2 10 Skills + L3 4 Workflows | Gate 2+3 | 公文處理 -40% |
| 4 | 19-24 | L4 市民服務 Agent + 全市推廣計畫 | Gate 4+4G | 1999 一線解決 +30% |

### Stage 7 解決方案架構

- **Variant C 全地端**：政府機房 + Llama 70B + Qdrant；零雲端
- 串接公文系統、1999 系統、SharePoint（皆內網）

### Stage 8 治理 ROI

- Permission：依局處 + 職等分層；公文 / 市民資料分級
- Audit：符合資安管理法；政風可稽核
- 三審流程：政風 + 法制 + 資安會簽
- ROI Tracker：公文處理時間、1999 解決率、跨局處問答使用率、市民滿意度

## 6. 診斷報告摘要

> **核心發現**：市政府數位局 G 流程標準化成熟（L2.5）但 AI 工具空白（L1）。公部門導入特性 — 採購與三審週期長，但標竿一旦建立可全市 22 局處複製。建議 24 個月先導 3 局處達 L3-L4，重點是可複製模式。
>
> **預期 ROI**：24 個月先導效益 NT$ 6,200 萬；全市推廣後年化效益估 NT$ 2.8 億。

## 7. 30/60/90 天 Roadmap

### 30 天
- 地端機房採購規格書（採購程序啟動）
- AI 治理規範草案（政風 + 法制 + 資安）
- 先導 3 局處確認 + AI Champion 指派

### 60 天
- 採購評選作業
- AI 治理規範三審
- L1 教材在地化（公部門場景）

### 90 天
- 機房建置啟動
- AI 治理規範核定
- L1 第一梯次（民政局）籌備

## 9. 風險與治理

| 風險 | 機率 | 影響 | 緩解 |
| --- | --- | --- | --- |
| 市民個資外流 | 低（地端） | 致命 | 全地端、零雲端、Audit |
| AI 公文錯誤造成行政疏失 | 中 | 高 | 承辦人 review 強制；Agent 僅輔助 |
| 採購程序延宕 | 高 | 中 | 提前 6 個月啟動；分年編列 |
| 跨局處協調困難 | 高 | 中 | 數位局長 + 研考會雙 sponsor；先導示範 |
| 公務員抗拒 / 工會疑慮 | 中 | 中 | 不裁員承諾；定位為「減負」非「取代」 |
| 政權更替影響延續性 | 中 | 高 | 制度化為常設規範；標竿成果文件化 |

## 10. 結果摘要

24 個月後：
- 先導 3 局處從 L1 推進至 L3-L4
- 公文處理時間 -40%；1999 一線解決率 55% → 85%
- 產出「市府 AI 導入標準作業模式」可供 22 局處複製
- 數位局成為其他縣市參訪標竿
- AI 治理規範納入市府常設制度
