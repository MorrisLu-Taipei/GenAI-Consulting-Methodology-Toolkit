# AI 成熟度問卷

更新日期：2026-05-15

## 1. 使用方式

本問卷提供三種版本：

- 10 題快速診斷：業務開發、初次會議、提案前使用。
- 25 題課前診斷：安排課程比例與案例時使用。
- 50 題顧問深度診斷：上課後進入八階段顧問診斷時使用。

每題採 0-4 分：

| 分數 | 說明 |
| --- | --- |
| 0 | 完全沒有 |
| 1 | 零星個人使用 |
| 2 | 部門有初步做法 |
| 3 | 已有標準做法並可重複 |
| 4 | 已制度化、可治理、可衡量 |

### 1.1 給填答者的話

這份問卷是設計給**所有部門主管**填的，不是只給 IT。**你不需要懂技術名詞**——每一題下面都有「白話解釋」，10 題版本還附了 0／2／4 分的「情境對照」，你只要看舉例、想想「我們公司比較像哪一種」，就能打分。

填答原則：

- **不確定就往低分打**。診斷的目的是找出真實落點，分數灌水只會讓後面的課程與導入計畫對不準。
- **看到看不懂的英文或縮寫**（OpenWebUI、n8n、CRM、ERP、API…），先翻下面的 §1.2 名詞小辭典。
- **沒有對錯**。這是健檢，不是考試。L1、L2 不是「差」，是「現在在哪一層、下一步往哪走」。

### 1.2 名詞小辭典

| 名詞 | 白話解釋 |
| --- | --- |
| OpenWebUI | 一個長得像 ChatGPT 的網頁介面，但公司可以自己架、自己管。員工登入就能用，IT 看得到誰在用、用來做什麼。 |
| 雲端 AI／公有雲 AI | ChatGPT、Gemini、Claude 這類由國外公司提供、資料會傳到對方伺服器處理的服務。 |
| Prompt（提示詞）| 你給 AI 的指令或問法。好的 Prompt 像「給新人的工作交辦單」，寫得清楚就能穩定拿到好結果。 |
| SOP | 標準作業流程，把「一件事該怎麼做」寫成固定步驟。 |
| Skill（技能包）| 把一個常做任務（例如「寫客訴回覆」）的指令、範例、步驟打包好，下次直接套用，不用每次重想。 |
| n8n | 一個流程自動化工具，用拖拉方式把不同軟體串起來，讓工作自動跑，像一條「數位生產線」。 |
| Workflow（工作流程自動化）| 一連串原本要人手動點來點去的步驟，設定好後讓系統自動完成。 |
| 自動觸發 | 不用人手動按開始；某個事件發生（收到信、表單送出、時間到）就自動開跑。 |
| AI Agent（AI 代理人）| 比聊天 AI 更進一步：你給一個目標，它會自己拆步驟、查資料、用工具，最後回報結果。像「會自己想辦法完成任務的數位助理」。 |
| Agent Team／多 Agent 協作 | 多個各有專長的 AI Agent 像一個小組分工合作（例如一個查市場、一個寫文案、一個算財務）。 |
| API／Webhook | 軟體之間互相傳資料的「接口」。有 API 代表這個系統可以被自動化串接；不確定就問 IT。 |
| 資料庫 | 公司存放結構化資料（客戶、訂單、庫存）的地方。 |
| CRM | 客戶關係管理系統，存客戶、商機、業務往來紀錄（如 Salesforce、HubSpot）。 |
| ERP | 企業資源規劃系統，管財務、進銷存、生產（如 SAP、Oracle、鼎新）。 |
| MES | 製造執行系統，管工廠現場生產。 |
| PLM | 產品生命週期管理，管產品設計與工程資料。 |
| QMS | 品質管理系統，管品檢、不良、稽核。 |
| 資料分級 | 把公司資料依機密程度分類（公開／內部／機密／極機密），決定哪些能給 AI、哪些不行。 |
| 雲 AI／Hybrid／全地端（部署模式）| 雲 AI＝直接用國外服務，最快但資料會出公司；全地端＝AI 裝在公司自己的機器，資料不出門但要 IT 維運；Hybrid＝一般資料用雲、機密資料留地端。 |
| Stage Gate（階段關卡）| 每個導入階段設一個驗收點，過關才往下走，避免做到一半失控。 |
| PoC（概念驗證）| 小規模先做一個看得到成果的範例，證明「這真的有用」再擴大。 |
| Log（執行紀錄）| 系統自動留下的「做了什麼、何時做、成功或失敗」的紀錄。 |
| 人工審核節點 | 流程中保留一個「人來確認」的關卡，不讓 AI 直接做高風險決定。 |
| KPI | 衡量成效的指標（例如省下多少時間、錯誤率降多少）。 |

## 2. 10 題快速診斷

> 每題 0-4 分（對照 §1 計分表）。每題附「白話解釋」與 0／2／4 分的「情境對照」，挑最接近你公司現況的描述打分即可；落在中間就打 1 或 3。

### Q1｜公司有沒有一個「官方核准、大家統一用」的 AI 入口？

- **構面**：工具使用
- **白話解釋**：問的是員工用 AI 時，是各自拿手機開免費 ChatGPT（公司完全管不到），還是公司有一個官方、統一、管得到的地方讓大家登入使用。
- **情境對照**：
  - **0 分**：完全沒有，員工各用各的，公司不知道誰在用、拿來做什麼。
  - **2 分**：有少數人共用一個 ChatGPT 付費帳號，但沒公告、沒規範。
  - **4 分**：公司架了統一入口（例如 OpenWebUI 這種自架的 ChatGPT 介面），全員有帳號、IT 看得到使用狀況，也訂了「哪些資料不能貼上去」的規範。

### Q2｜員工是否每週穩定用 AI 處理工作（文件、Email、摘要、分析、報告）？

- **構面**：工具使用
- **白話解釋**：問的是「用 AI」是少數人偶爾玩玩，還是已經變成多數同仁每週的工作習慣。
- **情境對照**：
  - **0 分**：幾乎沒人用，或只有老闆自己偶爾問問。
  - **2 分**：行銷、業務有幾個人會用 AI 寫文案、改信，但其他部門沒動靜。
  - **4 分**：跨部門多數同仁每週都會用 AI 做摘要、寫初稿、整理資料、分析數字，已經是日常。

### Q3｜部門有沒有把「好用的問法、做法」整理成可共用的資產？

- **構面**：知識沉澱
- **白話解釋**：問的是同仁的訣竅有沒有被整理下來變成公司資產，還是每個人的好用問法都鎖在自己腦袋裡。Skill＝把一個常做任務的指令和範例打包好、可重複套用。
- **情境對照**：
  - **0 分**：完全沒整理，每個人每次都重頭跟 AI 重講一遍。
  - **2 分**：有同仁私下存了一些好用的 Prompt，但沒有共用、沒有整理。
  - **4 分**：部門有共用的 Prompt 庫／SOP／模板／Skill，遇到新任務會先找有沒有現成的可套。

### Q4｜新人能不能靠公司現成的工具包，快速完成標準工作？

- **構面**：知識沉澱
- **白話解釋**：問的是新人或不熟的人，能不能靠公司現成的工具包快速上手標準工作，而不是一定要老手手把手帶。
- **情境對照**：
  - **0 分**：新人完全靠資深員工口頭教，沒文件、沒工具。
  - **2 分**：有一些 SOP 文件，但 AI 工具還是要自己摸索。
  - **4 分**：新人打開共用 Skill／文件就能完成大部分高頻標準工作，上手很快。

### Q5｜公司有沒有把 AI 跟其他系統「接起來」自動讀寫資料？

- **構面**：系統整合
- **白話解釋**：問的是 AI 有沒有跟公司其他系統串接、能自動讀寫資料，而不是只能在對話框裡單獨使用。（Gmail＝信箱；Sheets／Excel＝試算表；Notion／CRM／ERP＝公司用的各種系統，看不懂可翻名詞小辭典）
- **情境對照**：
  - **0 分**：AI 完全獨立，要它處理資料只能人工複製貼上。
  - **2 分**：試過用工具把 AI 接上 Google Sheets 或信箱，做過一兩個小實驗。
  - **4 分**：已有 AI／n8n 穩定串接信箱、試算表、CRM、ERP 等系統，自動讀寫資料。

### Q6｜公司有沒有「設定好就會自己跑」的自動化流程？

- **構面**：流程自動化
- **白話解釋**：問的是有沒有「設定好就自己會跑」的流程——某件事發生後系統自動接著做、做完通知、並留下紀錄，不用人一步步盯。
- **情境對照**：
  - **0 分**：所有流程都要人手動執行，AI 幫不上自動化。
  - **2 分**：做過一兩個自動化小流程，但不穩定、常要人介入。
  - **4 分**：已有流程能自動觸發、自動執行、自動通知相關人、並留下執行紀錄（Log）。

### Q7｜公司有沒有用到「會自己執行任務」的 AI Agent？

- **構面**：Agent 應用
- **白話解釋**：聊天 AI 是「你問一句、它答一句」；AI Agent 更進一步——你給一個目標，它會自己拆步驟、自己去用工具查資料，最後把結果回報給你。問的是公司有沒有用到這種。
- **情境對照**：
  - **0 分**：只有聊天問答，沒有會自己執行任務的 Agent。
  - **2 分**：試做過簡單 Agent（例如自動整理週報），還在實驗階段。
  - **4 分**：已有 Agent 穩定運作，能拆解任務、呼叫工具、回報結果，且有人工接手機制。

### Q8｜公司有沒有試過讓多個 AI Agent 分工協作？

- **構面**：Agent 應用
- **白話解釋**：再進一步——不是一個 Agent，而是好幾個各有專長的 AI Agent 像一個小組分工合作。問的是公司有沒有試過這種「AI 團隊」。
- **情境對照**：
  - **0 分**：完全沒有，連單一 Agent 都還沒有。
  - **2 分**：正在規劃，或在小範圍實驗多個 Agent 分工。
  - **4 分**：已實際讓多個 Agent（如市場、產品、行銷、客服、財務）協作完成跨部門任務。

### Q9｜用 AI 有沒有「踩煞車的機制」（資料分級、權限、人工審核、使用紀錄）？

- **構面**：治理與 ROI
- **白話解釋**：問的是用 AI 有沒有「踩煞車的機制」——哪些資料能給 AI／哪些不能、誰能用、高風險決定有沒有人複核、AI 做了什麼有沒有留紀錄。
- **情境對照**：
  - **0 分**：完全沒有，員工想貼什麼給 AI 都行，沒人管也沒紀錄。
  - **2 分**：口頭講過「機密資料不要貼」，但沒有制度、沒有紀錄。
  - **4 分**：有資料分級、權限控管、高風險動作有人工審核、AI 使用有紀錄可稽核。

### Q10｜管理層有沒有用「數字」定義 AI 導入成不成功？

- **構面**：治理與 ROI
- **白話解釋**：問的是老闆／主管有沒有用「數字」來定義 AI 導入算不算成功，還是只是「感覺有用」。KPI＝衡量成效的指標。
- **情境對照**：
  - **0 分**：沒有任何指標，導入 AI 全憑感覺。
  - **2 分**：主管心裡有期待（希望快一點、省一點），但沒有寫成具體數字。
  - **4 分**：管理層已定義明確 KPI，例如「每月省 X 小時」「錯誤率降 Y%」「回應速度快 Z%」，並會定期追蹤。

### 快速判讀

- 0-10 分：L0-L1，需先建立 AI 使用入口與基礎訓練。
- 11-20 分：L1，已有個人使用，需開始 Skill 化。
- 21-28 分：L2，部門能力開始沉澱，適合推 Workflow。
- 29-34 分：L3，已有流程基礎，適合導入 Agent。
- 35-40 分：L4-L5，可評估 Agent Team 與治理擴大。

## 3. 25 題課前診斷

> 每題 0-4 分，總分 100 分。每題附「白話＋例如」一行說明；看不懂的名詞請翻 §1.2 名詞小辭典。

### A. 工具使用

**A1｜公司是否有統一或核准的 AI 工具入口？**
→ **白話**：員工用 AI 是各用各的，還是公司有官方、管得到的統一入口。**例如**：已架 OpenWebUI 或統一發企業帳號＝高分；大家各自用免費版、公司管不到＝低分。

**A2｜員工是否知道哪些資料可以、哪些不可以輸入 AI？**
→ **白話**：同仁心裡有沒有一條「紅線」，知道客戶個資、財報、合約這類機密不能隨便貼給 AI。**例如**：有書面規範並教育過＝高分；從沒講過、各憑良心＝低分。

**A3｜員工是否能用 AI 完成摘要、改寫、翻譯、分類、報告初稿？**
→ **白話**：問的是實際動手能力，不是「知道 AI 可以」。**例如**：多數同仁能用 AI 把長文件變摘要、把草稿改順、寫出報告初稿＝高分。

**A4｜是否已有部門或公司級的 AI 使用規範？**
→ **白話**：有沒有白紙黑字的規則（能用什麼、不能做什麼、出事找誰）。**例如**：有一頁式使用守則並公告全員＝高分；只靠口頭提醒＝低分。

**A5｜是否能舉出具體的「AI 提升個人效率」案例？**
→ **白話**：能不能講出真實故事，而不是空泛說「AI 很好用」。**例如**：「採購用 AI 比價、整理報價，每週省 3 小時」＝高分。

### B. 知識沉澱

**B1｜部門是否有共用的 Prompt 或任務模板？**
→ **白話**：好用的「問法」有沒有變成大家共用的資產。**例如**：部門有一份共用 Prompt 清單、新人直接拿來用＝高分；訣竅鎖在個人腦袋＝低分。

**B2｜是否把資深員工的經驗整理成 SOP、Checklist 或 Skill？**
→ **白話**：老手的經驗有沒有被「寫下來、打包起來」讓別人也能用。**例如**：把資深客服的回覆訣竅整理成 Skill＝高分。

**B3｜是否有集中管理 Skill 的固定位置？**
→ **白話**：這些整理好的工具包有沒有一個固定的家（共用資料夾、Notion、Git），還是散落各處找不到。**例如**：有統一知識庫、大家知道去哪找＝高分。

**B4｜新人是否能直接套用 Skill 完成高頻工作？**
→ **白話**：新人靠現成工具包能不能快速完成常做的工作。**例如**：新人第一週就能用共用 Skill 產出標準文件＝高分。

**B5｜Skill 是否有負責人、版本與品質檢查機制？**
→ **白話**：這些工具包有沒有人維護、有沒有版本、有沒有人把關品質，避免過時或出錯。**例如**：每個 Skill 有 owner 和更新紀錄＝高分。

### C. 流程自動化

**C1｜是否盤點過「哪些高頻流程可以自動化」？**
→ **白話**：有沒有坐下來盤點過「哪些工作天天做、最適合交給系統自動跑」。**例如**：列過一份自動化候選清單＝高分。

**C2｜是否已用 n8n 或類似工具串接工作流程？**
→ **白話**：有沒有實際用流程自動化工具（n8n＝用拖拉方式把軟體串起來自動跑）做出東西。**例如**：已有上線、天天在跑的 n8n 流程＝高分。

**C3｜是否有流程能自動觸發、執行、通知與紀錄？**
→ **白話**：流程是不是「自己會跑」——事件一發生就自動做、做完通知、留紀錄。**例如**：收到客訴信→自動分類→通知負責人→記錄＝高分。

**C4｜是否有「人工審核節點」避免 AI 直接做高風險決策？**
→ **白話**：高風險的事（退款、對外發信、財務決定）有沒有保留「人來按確認」的關卡。**例如**：AI 擬好、人核可才送出＝高分。

**C5｜是否有錯誤通知、自動重試與執行紀錄（Log）？**
→ **白話**：流程出錯時，系統會不會通知人、會不會自動重試、有沒有留下「哪裡出錯」的紀錄。**例如**：失敗自動發通知＋自動重試＝高分。

### D. 系統整合

> 這一組在盤點「AI 將來要接哪些系統」，照公司實際情況回答即可，不確定就問 IT 或資訊窗口。

**D1｜是否需要串接 Gmail 或企業信箱？**
→ **白話**：公司有沒有大量靠 Email 的工作要交給 AI 幫忙。**例如**：客服、業務天天處理大量信件＝需要。

**D2｜是否需要串接 Google Sheets、Excel 或報表系統？**
→ **白話**：很多公司資料其實放在試算表，AI 要用就得接上。**例如**：報價單、追蹤表、KPI 表都在 Excel／Sheets＝需要。

**D3｜是否需要串接 Notion、SharePoint、Confluence 或知識庫？**
→ **白話**：問公司的文件、知識放在哪些平台，AI 要查就得接。看不懂這些名稱就想「我們公司的文件平台是什麼」。

**D4｜是否需要串接 CRM、ERP、MES、PLM、QMS 或內部系統？**
→ **白話**：問公司的核心營運系統（客戶、財務進銷存、工廠生產、產品設計、品質——看不懂可翻名詞小辭典）。**例如**：業務流程要 AI 幫忙就要接 CRM；製造排程要幫忙就要接 ERP／MES。

**D5｜是否有 API、資料庫或 Webhook 可供 AI 流程使用？**
→ **白話**：白話就是「這些系統打不打得開、能不能讓別的軟體自動讀寫」。不確定就問 IT——有 API＝能自動串接＝高分。

### E. Agent 應用

**E1｜是否已有明確的 Agent 任務？**
→ **白話**：有沒有想清楚要讓 AI Agent 做哪件具體的事，而不是空泛說「想做 Agent」。**例如**：「每週一自動整理上週業績週報」就是明確任務。

**E2｜Agent 是否能呼叫 Skill 或 n8n Workflow？**
→ **白話**：Agent 是不是真的「手上有工具」可以用，還是只能空想。**例如**：Agent 能自己去跑一個 n8n 流程查資料＝高分。

**E3｜Agent 的權限、資料來源與可執行動作是否清楚？**
→ **白話**：有沒有明確界定 Agent「能碰什麼資料、能做什麼動作、不能做什麼」。**例如**：寫清楚「Agent 只能讀、不能刪改」＝高分。

**E4｜Agent 是否有回報格式、異常處理與人工接手機制？**
→ **白話**：Agent 做完怎麼回報、出狀況怎麼辦、卡住時人能不能接手。**例如**：固定格式回報＋異常時自動轉人工＝高分。

**E5｜是否有多 Agent 協作的需求或實驗？**
→ **白話**：有沒有想過或試過讓好幾個 AI Agent 分工合作。**例如**：規劃過「市場 Agent＋文案 Agent＋財務 Agent」一起做提案＝高分。

### 25 題診斷輸出

每題 0-4 分，總分 100 分。

輸出欄位：

- 總分。
- 六大構面分數。
- 初判成熟度。
- 建議課程比例。
- 建議部署模式。
- 建議優先 PoC。

## 4. 50 題顧問深度診斷

50 題版本在 25 題基礎上，加入公司屬性、部署模式與 ROI。此版本通常由 IT 或 AI 窗口協助填寫；以下每組都附簡短說明。

### F. 公司屬性

> 這一組是了解公司基本背景，照實回答即可，沒有對錯。

| 題號 | 題目 |
| --- | --- |
| F1 | 公司主要產業是研發製造、行銷服務、金融、醫療、政府、教育、零售、物流、軟體或其他？ |
| F2 | 哪些部門最需要 AI？ |
| F3 | 哪些工作最重複、最耗時、最容易出錯？ |
| F4 | 哪些工作最依賴資深員工的個人經驗？ |
| F5 | 管理層最重視效率、成本、品質、營收、創新、治理或資安？ |

### G. 部署模式

> 這一組在判斷 AI 該裝在哪裡（雲端／公司自己的機器／混合）。看不懂的名詞請翻 §1.2。

| 題號 | 題目 | 白話與舉例 |
| --- | --- | --- |
| G1 | 公司是否允許使用公有雲 AI 處理一般工作資料？ | 公有雲 AI＝ChatGPT、Gemini 這類資料會傳到國外公司的服務。問的是公司政策允不允許用它處理一般（非機密）工作。 |
| G2 | 是否有核心資料不得離開公司內網？ | 有沒有哪些資料規定「只能留在公司內部、不能上任何雲端」。 |
| G3 | 是否有個資、研發、財務、合約、客戶機密或法規資料？ | 盤點公司有沒有這些敏感資料——有的話部署模式就要更謹慎。 |
| G4 | 是否已有資料分級制度？ | 有沒有把資料依機密程度分類（公開／內部／機密）。例如：文件都有標密級＝高分。 |
| G5 | 是否有內部 IT 團隊可維護伺服器、Docker、API、資料庫與權限？ | 白話就是「公司有沒有自己的技術人手，養得起一套裝在自家機器上的系統」。沒有的話就偏向用雲端服務。 |

### H. 系統盤點

> 這一組是清點公司現有系統，請填實際的系統名稱；不確定就問 IT 或資訊窗口。

| 題號 | 題目 | 白話與舉例 |
| --- | --- | --- |
| H1 | 主要協作工具是 Google Workspace、Microsoft 365、Notion、SharePoint 或其他？ | 就是公司平常收信、開會、共用文件用的那一套。 |
| H2 | 是否使用 CRM？系統名稱是什麼？ | CRM＝管客戶與業務的系統（如 Salesforce、HubSpot）。 |
| H3 | 是否使用 ERP / MES / PLM / QMS？系統名稱是什麼？ | 分別是管財務進銷存／工廠生產／產品設計／品質的系統，看不懂可翻名詞小辭典。 |
| H4 | 是否有內部 API 或資料庫可讀取？ | 白話：問 IT「我們的系統能不能讓別的程式自動讀資料」。 |
| H5 | 是否有權限、Credential、帳號與稽核管理機制？ | Credential＝系統的帳號密碼／金鑰。問的是這些有沒有集中管理、有沒有稽核紀錄。 |

### I. ROI 與治理

> 這一組在看「AI 導入有沒有人負責、有沒有在算成效」。

| 題號 | 題目 | 白話與舉例 |
| --- | --- | --- |
| I1 | 是否能估算每個 AI 場景節省多少時間？ | 能不能講出「這件事本來要 X 小時，現在 Y 小時」。 |
| I2 | 是否能追蹤錯誤率、回應速度、處理量或品質改善？ | 有沒有在用數字看 AI 帶來的改變。 |
| I3 | 是否有 AI 專案 Owner、IT Owner 與部門 Owner？ | 有沒有明確「誰負責推動、誰負責技術、誰負責業務面」。 |
| I4 | 是否有 Stage Gate 驗收標準？ | Stage Gate＝每個階段設一個驗收關卡，過關才往下做。 |
| I5 | 是否定期回顧 AI 導入成效並調整 Roadmap？ | 有沒有定期檢討、按結果修正計畫。 |

## 5. 部署模式判讀題

可搭配 25 題或 50 題使用。逐列勾選符合公司現況的描述，把對應分數加總，最高分的那一欄就是建議的部署模式。

| 題目 | 雲 AI | Hybrid | 全地端 |
| --- | ---: | ---: | ---: |
| 資料敏感度低 | +2 | +1 | 0 |
| 部分資料敏感 | 0 | +2 | +1 |
| 核心資料不得上雲 | 0 | +1 | +2 |
| IT 維運能力有限 | +2 | +1 | 0 |
| 有 IT 與資安團隊 | 0 | +1 | +2 |
| SaaS 工具為主 | +2 | +1 | 0 |
| ERP / MES / 內部系統為主 | 0 | +1 | +2 |
| 預算有限，要快速開始 | +2 | +1 | 0 |
| 高稽核或法規要求 | 0 | +1 | +2 |

最高分為建議部署模式。若 Hybrid 與全地端接近，建議從 Hybrid 試點開始，核心資料仍留在地端。

## 6. 診斷結果文字模板

### L1 範例

貴公司目前 AI 應用主要落在 L1 Chat AI 階段。員工已開始使用 AI 提升個人效率，但知識尚未沉澱為部門資產。建議先建立 OpenWebUI 統一入口、AI 使用規範與部門高頻 Prompt，再進入 L2 Skill 化。

### L2 範例

貴公司已具備初步 L2 Skill AI 基礎。部門有部分 SOP、Prompt 或模板，但尚未形成穩定可維護的 Skill Library。建議課程聚焦 Skill 設計、部門知識沉澱與 L3 Workflow 候選場景盤點。

### L3 範例

貴公司已有 L3 Workflow AI 條件。下一步應挑選高價值流程，透過 n8n 串接 Gmail、Sheets、Notion、CRM、API、ERP 等系統，完成 1-2 個可展示 PoC，並建立權限、Log 與人工審核機制。

### L4 範例

貴公司已有導入 Hermes Agent 的基礎。建議從單一部門任務開始，設計 Agent 角色卡、任務邊界、可呼叫 Skill、可啟動 Workflow 與回報格式，避免一開始就讓 Agent 處理高風險決策。

### L5 範例

貴公司具備探索 ClawTeam 的條件。建議選擇跨部門任務，例如新產品規劃、客戶經營、營運異常分析，設計多 Agent 角色分工、輸出整合與品質檢查流程，讓 AI Team 成為管理決策支援能力。
