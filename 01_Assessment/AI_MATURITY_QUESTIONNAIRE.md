# AI 成熟度問卷

更新日期：2026-05-13

## 1. 使用方式

本問卷提供三種版本：

- 10 題快速診斷：業務開發、初次會議、提案前使用。
- 25 題課前診斷：安排課程比例與案例時使用。
- 50 題顧問深度診斷：上課後進入八階段顧問診斷時使用。

每題採 0-4 分：

| 分數 | 說明 |
| --- | --- |
| 0 | 完全沒有 |
| 1 | 零星個人使用 |
| 2 | 部門有初步做法 |
| 3 | 已有標準做法並可重複 |
| 4 | 已制度化、可治理、可衡量 |

## 2. 10 題快速診斷

| 題號 | 題目 | 構面 |
| --- | --- | --- |
| Q1 | 公司是否已有核准的 AI 使用入口，例如 OpenWebUI 或指定雲端 AI 工具？ | 工具使用 |
| Q2 | 員工是否每週穩定使用 AI 處理文件、Email、摘要、分析或報告？ | 工具使用 |
| Q3 | 部門是否已整理可共用的 Prompt、SOP、模板或 Skill？ | 知識沉澱 |
| Q4 | 新人是否能透過共用 Skill 或文件快速完成標準工作？ | 知識沉澱 |
| Q5 | 公司是否已有 AI 或 n8n 串接 Gmail、Sheets、Notion、CRM、API、ERP 等系統？ | 系統整合 |
| Q6 | 是否已有流程可以被自動觸發、執行、通知與紀錄？ | 流程自動化 |
| Q7 | 是否已有 AI Agent 能拆解任務、呼叫工具並回報結果？ | Agent 應用 |
| Q8 | 是否已嘗試多 Agent 協作，例如市場、產品、行銷、客服、財務 Agent 分工？ | Agent 應用 |
| Q9 | 是否有資料分級、權限控管、人工審核或 AI 使用紀錄？ | 治理與 ROI |
| Q10 | 管理層是否已定義 AI 導入 KPI，例如時間節省、錯誤率、回應速度、成本或營收？ | 治理與 ROI |

### 快速判讀

- 0-10 分：L0-L1，需先建立 AI 使用入口與基礎訓練。
- 11-20 分：L1，已有個人使用，需開始 Skill 化。
- 21-28 分：L2，部門能力開始沉澱，適合推 Workflow。
- 29-34 分：L3，已有流程基礎，適合導入 Agent。
- 35-40 分：L4-L5，可評估 Agent Team 與治理擴大。

## 3. 25 題課前診斷

### A. 工具使用

| 題號 | 題目 |
| --- | --- |
| A1 | 公司是否有統一或核准的 AI 工具入口？ |
| A2 | 員工是否知道哪些資料可以或不可以輸入 AI？ |
| A3 | 員工是否能使用 AI 完成摘要、改寫、翻譯、分類、報告初稿？ |
| A4 | 是否已有部門或公司級 AI 使用規範？ |
| A5 | 是否能舉出具體 AI 個人效率提升案例？ |

### B. 知識沉澱

| 題號 | 題目 |
| --- | --- |
| B1 | 部門是否有共用 Prompt 或任務模板？ |
| B2 | 是否把資深員工經驗整理成 SOP、Checklist 或 Skill？ |
| B3 | 是否有集中管理 Skill 的位置，例如 Notion、Git、文件庫？ |
| B4 | 新人是否能直接套用 Skill 完成高頻工作？ |
| B5 | 是否有 Skill 的負責人、版本與品質檢查機制？ |

### C. 流程自動化

| 題號 | 題目 |
| --- | --- |
| C1 | 是否盤點過可自動化的高頻流程？ |
| C2 | 是否已有 n8n 或類似工具串接工作流程？ |
| C3 | 是否有流程能自動觸發、執行、通知與紀錄？ |
| C4 | 是否有人工審核節點避免 AI 直接做高風險決策？ |
| C5 | 是否有錯誤通知、重試與執行 Log？ |

### D. 系統整合

| 題號 | 題目 |
| --- | --- |
| D1 | 是否需要串接 Gmail 或企業信箱？ |
| D2 | 是否需要串接 Google Sheets、Excel 或報表系統？ |
| D3 | 是否需要串接 Notion、SharePoint、Confluence 或知識庫？ |
| D4 | 是否需要串接 CRM、ERP、MES、PLM、QMS 或內部系統？ |
| D5 | 是否有 API、資料庫或 Webhook 可供 AI Workflow 使用？ |

### E. Agent 應用

| 題號 | 題目 |
| --- | --- |
| E1 | 是否已有明確的 Agent 任務，例如整理週報、分析異常、產出提案？ |
| E2 | Agent 是否能呼叫 Skill 或 n8n Workflow？ |
| E3 | Agent 的權限、資料來源與可執行動作是否清楚？ |
| E4 | Agent 是否有回報格式、異常處理與人工接手機制？ |
| E5 | 是否有多 Agent 協作的需求或實驗？ |

### 25 題診斷輸出

每題 0-4 分，總分 100 分。

輸出欄位：

- 總分。
- 六大構面分數。
- 初判成熟度。
- 建議課程比例。
- 建議部署模式。
- 建議優先 PoC。

## 4. 50 題顧問深度診斷

50 題版本在 25 題基礎上，加入公司屬性、部署模式與 ROI。

## 5. 部署模式判讀題

可搭配 25 題或 50 題使用。

| 題目 | 雲 AI | Hybrid | 全地端 |
| --- | ---: | ---: | ---: |
| 資料敏感度低 | +2 | +1 | 0 |
| 部分資料敏感 | 0 | +2 | +1 |
| 核心資料不得上雲 | 0 | +1 | +2 |
| IT 維運能力有限 | +2 | +1 | 0 |
| 有 IT 與資安團隊 | 0 | +1 | +2 |
| SaaS 工具為主 | +2 | +1 | 0 |
| ERP / MES / 內部系統為主 | 0 | +1 | +2 |
| 預算有限，要快速開始 | +2 | +1 | 0 |
| 高稽核或法規要求 | 0 | +1 | +2 |

最高分為建議部署模式。若 Hybrid 與全地端接近，建議從 Hybrid 試點開始，核心資料仍留在地端。

## 6. 診斷結果文字模板

### L1 範例

貴公司目前 AI 應用主要落在 L1 Chat AI 階段。員工已開始使用 AI 提升個人效率，但知識尚未沉澱為部門資產。建議先建立 OpenWebUI 統一入口、AI 使用規範與部門高頻 Prompt，再進入 L2 Skill 化。

### L2 範例

貴公司已具備初步 L2 Skill AI 基礎。部門有部分 SOP、Prompt 或模板，但尚未形成穩定可維護的 Skill Library。建議課程聚焦 Skill 設計、部門知識沉澱與 L3 Workflow 候選場景盤點。

### L3 範例

貴公司已有 L3 Workflow AI 條件。下一步應挑選高價值流程，透過 n8n 串接 Gmail、Sheets、Notion、CRM、API、ERP 等系統，完成 1-2 個可展示 PoC，並建立權限、Log 與人工審核機制。

### L4 範例

貴公司已有導入 Hermes Agent 的基礎。建議從單一部門任務開始，設計 Agent 角色卡、任務邊界、可呼叫 Skill、可啟動 Workflow 與回報格式，避免一開始就讓 Agent 處理高風險決策。

### L5 範例

貴公司具備探索 ClawTeam 的條件。建議選擇跨部門任務，例如新產品規劃、客戶經營、營運異常分析，設計多 Agent 角色分工、輸出整合與品質檢查流程，讓 AI Team 成為管理決策支援能力。
