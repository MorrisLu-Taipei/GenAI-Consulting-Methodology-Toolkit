## 1. Hero Tagline / 主標

> **企業 AI 轉型不是上工具，是建立可驗證的成熟度。**
> **Enterprise AI transformation is not about adopting tools — it is about building verifiable maturity.**

副標：用 L1-L5 五層成熟度模型，把 AI 從個人使用一路推進到 Agent Team 自主協作。
Subtitle: Use the L1-L5 maturity model to evolve AI from individual usage to autonomous Agent Team collaboration.

## 2. 三大痛點 / Problem Statement

- 員工用 AI 沒有共識：每個人開不同帳號、共用密碼、機敏資料外流。
  No alignment on AI usage: shared accounts, password sharing, leaked sensitive data.
- 導入工具但沒有方法論：買了 ChatGPT / Copilot 但無從衡量成效。
  Tools without methodology: licenses bought, but no measurable outcomes.
- 試了 RAG / Agent 但無法擴展：PoC 不能變成生產級流程。
  Failed-to-scale RAG / Agent pilots: PoCs that never become production workflows.

## 3. 三段式服務流程 / Three-Step Service Flow

1. **診斷 Diagnose** — 10/25/50 題 AI 成熟度問卷 + 公司屬性調查 → 客觀 L1-L5 分數
2. **建置 Build** — L1-L5 課程依分數配比、課中產出可驗收資產 (Prompt / Skill / Workflow / Agent / Team)
3. **交付 Deliver** — 八階段顧問診斷報告 + 30/60/90 天 Roadmap + ROI 與治理建議

`Diagnose → Build → Deliver` — 6 to 12 weeks per engagement.

## 4. L1-L5 成熟度地圖 / L1-L5 Maturity Map (text representation)

| Level | Name | Tool | One-line value |
| --- | --- | --- | --- |
| **L1** | Chat AI | OpenWebUI | 全員安全使用 AI；累積 Prompt 與情境 |
| **L2** | Skill AI | Antigravity / Claude Code / Codex | 把個人經驗封裝為可複用 Skill |
| **L3** | Workflow AI | n8n | Skill 接上 Gmail/Sheets/Notion/CRM/ERP 變成 Workflow |
| **L4** | Auto Agentic AI | Hermes Agent | Wiki + Skill + Workflow + Gate = 可驗證 Agent |
| **L5** | Agentic Team AI | ClawTeam | 多個專業 Agent 組成 Team 完成跨部門任務 |

> 上一層產出 = 下一層輸入。
> Each layer's output is the next layer's input.

## 5. 可驗證交付物 (節錄) / Verifiable Deliverables (selected)

- AI 成熟度問卷與評分結果 / Maturity score & radar chart
- Skill Library + Antigravity artifacts
- n8n Workflow PoC + Execution Log
- Hermes Agent 任務卡 + Gate 4A-4E 驗收
- ClawTeam Agent Team 角色卡與整合報告
- 八階段顧問診斷報告 + 30/60/90 天 Roadmap
