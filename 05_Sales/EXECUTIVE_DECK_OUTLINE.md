> "80% of enterprise AI failures aren't about tools — they're about **skipping maturity levels**.

> What the consultant does is turn this reasoning into an auditable closed loop — once clients sign their Ideal Practice, **Gap = (your signed target − your As-Is); both numerator and denominator are your words**.

> **95% of clients continue** — because **they themselves** were shocked by the empty radar cells, and **they themselves** signed the desired state.
> This is scientific management — clients don't trust the consultant; they trust their own numbers."

## Slide 3 — 我們的洞察 / Our Insight

**標題:** AI 導入失敗不是工具問題，是成熟度問題
AI Adoption Fails Not Because of Tools, but Because of Maturity

**關鍵訊息:** 工具堆疊 ≠ 組織能力。沒有 L1 全員使用，後面 L2-L5 都會跳級失敗。
A tool stack is not organizational capability. Without L1 org-wide adoption, L2-L5 will fail by skipping levels.

**內容:**
- 80% 的企業 AI PoC 卡在「個別英雄使用」階段
- L1 不打底 → L2 Skill 無內容
- L2 沒 Skill → L3 Workflow 空管子
- L3 沒 Workflow → L4 Agent 無工具
- L4 沒 Agent → L5 Team 無成員

**視覺:** 五層金字塔 + 跳級失敗的紅色 ❌

**講者備註:** 這頁是讓客戶「啊原來如此」的關鍵。要讓他承認自己過去買工具沒效果是因為跳級。

## Slide 4 — 方法論 / Our Methodology

**標題:** L1-L5 五層 + 八階段顧問
L1-L5 Five Layers × Eight-Stage Consulting

**關鍵訊息:** 兩個正交框架：縱向 L1-L5 是能力；橫向八階段是診斷。
Two orthogonal frameworks: L1-L5 (vertical capability) × eight-stage consulting (horizontal diagnostic).

**內容:**
- L1 Chat / L2 Skill / L3 Workflow / L4 Agent / L5 Team
- 八階段：診斷 → 願景 → 戰略 → 路徑圖 → 設計 → 導入 → 治理 → 永續
- 兩者交叉產出：可驗證交付物 + Stage Gate

**視覺:** 矩陣圖 (5×8) — 每格 = 一個交付物

**講者備註:** 強調這是學術背景 (Prof. Michael Rosemann QUT) 與實務 (Tiger AI) 結合。

## Slide 6 — ROI 模型 / ROI Model

**標題:** 三類 ROI 槓桿 / Three ROI Levers

**關鍵訊息:** 時間節省 + 質量提升 + 風險下降，每一項都可量化。
Time saved + quality improved + risk reduced — all quantifiable.

**內容:**
- **時間 Time**：白領平均週 8-12 hr 可釋出（會議、信件、報表）
- **質量 Quality**：客服 FCR 上升 / 業務提案 win rate 上升 / 報表錯誤下降
- **風險 Risk**：客訴漏接下降 / 合規檢核覆蓋率上升 / 機敏外流下降

實際案例：客戶 A 業務提案 6h → 2h；客戶 B 客服處理量 +130%；客戶 C 主管行政時間 -35%。

**視覺:** 三圓圈交集 (Time / Quality / Risk) + 案例數字

**講者備註:** 用具體數字，不用百分比堆疊。實際案例請見 04_Scenarios/。

## Slide 7 — 治理與安全 / Governance & Security

**標題:** 為什麼董事會應該安心 / Why the Board Can Sleep at Night

**關鍵訊息:** L1 即內建治理：帳號、權限、模型控管、資料邊界、Audit Log。
Governance is built in from L1: accounts, permissions, model control, data boundaries, audit logs.

**內容:**
- 雲端 / Hybrid / 全地端 三種部署模式對應不同產業 (金融、醫療、政府)
- OpenWebUI 帳號 + 群組 + 權限矩陣
- 模型可見性控管 (哪些員工能用 GPT-4 / Claude / 本地 Llama)
- 機敏資料 redact / 不外送 / 在 LLM 前過濾
- Audit log + Stage Gate

**視覺:** 安全三層架構圖 (人/流程/資料)

**講者備註:** 對 CIO / 法遵很重要，務必停留 30 秒以上。

## Slide 9 — 三階段方案與報價 / Engagement Tiers

**標題:** 三種啟動方式 / Three Ways to Start

**關鍵訊息:** 半日體驗 → 一日工作坊 → 二日導入營 → 顧問診斷專案。
Half-day taster → 1-day workshop → 2-day bootcamp → consulting diagnostic project.

**內容 (請依實際報價調整):**
- **半日體驗** (4 hr)：診斷 + L1-L5 概念 → 適合決策評估
- **一日工作坊** (8 hr)：診斷 + L1 + L2 啟動 → 適合 5-10 人核心團隊
- **二日導入營** (16 hr)：診斷 + L1-L3 + 1 個 PoC → 適合部門級
- **顧問診斷專案** (6-12 週)：完整 L1-L5 + 八階段報告 → 適合全公司

報價請於提案階段個別討論 / Pricing discussed in proposal phase.

**視覺:** 階梯圖，由淺入深

**講者備註:** 不要在這頁鎖死價格，留彈性。
