# 05 Sales — 대외 영업 소재

> 🌐 中文 / Chinese: [README.md](README.md) ｜ English: [README_EN.md](README_EN.md)

## 1. 본 디렉터리의 위치

본 디렉터리는 방법론을 **비즈니스로 전환** 하는 첫 번째 조각: **대외 영업 소재**.

`01`-`03` 는 컨설팅 방법론 본체 (매우 강하지만, 「강함」 만으로는 안건으로 전환되지 않음); 본 디렉터리가 해결하는 문제: **잠재 고객이 아직 진단 프로세스에 진입하기 전 — 영업이 어떻게 그를 「사고 싶다」 고 만드는가?** 본 디렉터리는 One-Pager DM, 3 가지 길이의 덱 아웃라인, 비주얼 소재 사양, 20 문항 영업 FAQ 까지 제공 — 영업이 Lead 개발, 제안, 반론 처리 각 단계에서 기성·일관·전문적인 소재를 가질 수 있도록.

Engagement Lifecycle 의 **Sales 단계** 에 대응: 낯선 Lead 를 「설문 작성하고 진단 진입할 의향」 까지 인도.

본 디렉터리를 쓰는 사람: 영업 (Lead Gen / Sales Rep / Closer), 컨설턴트 (고객 C-Level 에 방법론 브리핑), 디자이너 (사양에 따라 실제 비주얼 제작).

## 2. 방법론 내 위치

| 축 | 매핑 |
| --- | --- |
| 3 단계 서비스 플로우 | **진단 전의 「프리세일」** — 고객을 진단 진입 의향까지 인도 |
| 8 단계 컨설팅 구조 | 8 단계에 대응하지 않음 (8 단계는 계약 후) |
| L1-L5 성숙도 | 영업 소재에서 L1-L5 모델을 셀링 포인트로 사용 |
| Engagement Lifecycle | **Sales — Lead Qualification / Discovery / Proposal** |

## 3. 목표 & 효과

| 목표 | 효과 |
| --- | --- |
| 일관·전문적인 대외 소재 | 다른 영업이 방법론을 통일적으로 말함, 브랜드 프로 |
| 3 가지 덱 길이로 다른 상황 대응 | C-Level 10 페이지, 표준 20 페이지, 방법론 30 페이지, 각각 적합 |
| 20 문항 영업 FAQ | 영업이 반론에 표준 답변 가짐, 즉흥 답변 없음 |
| 비주얼 소재 사양 | 디자이너가 명확한 브리프 가짐, 일관된 비주얼 자산 |
| One-Pager DM | 고객과의 첫 접촉에서 이미 전문적인 남기는 것 |

**본 디렉터리 건너뛰면**: 각 영업이 자체 슬라이드 제작, 화법 불일치, 반론을 즉흥 대응, 고객의 방법론 첫인상이 산만.

## 4. 워크플로 & 로직

```text
Lead 개발
   → ONE_PAGER_OUTLINE (One-Pager DM, 첫 접촉 남기는 것)
첫 정식 회의
   → EXECUTIVE_DECK_OUTLINE (10 페이지 C-Level 판, 긴급성 + 방법론 신뢰성 확립)
심층 제안
   → STANDARD_SALES_DECK_OUTLINE (20 페이지, 부서장/IT/HR 용)
   → CONSULTING_METHODOLOGY_DECK_OUTLINE (30 페이지, 방법론 깊이, CIO/전략장/학술 동료용)
반론 처리 (전 과정)
   → SALES_FAQ (20 문항 표준 Q&A)
실제 비주얼 제작
   → VISUAL_ASSETS_SPEC (디자이너가 사양에 따라 PNG/SVG/인쇄 파일 제작)
```

| 단계 | 누가 | 언제 | 입력 | 출력 |
| --- | --- | --- | --- | --- |
| One-Pager DM 발송 | 영업 (Lead Gen) | Lead 개발 | one-pager | 고객 초기 인식 |
| C-Level 브리핑 | 영업 / 컨설턴트 | 첫 회의 | 10 페이지 덱 | 고객이 심층 논의 의향 |
| 심층 제안 | Closer | 제안 단계 | 20/30 페이지 덱 | 고객이 SOW 서명 의향 |
| 반론 처리 | 영업 / Closer | 전 과정 | SALES_FAQ | 고객 우려 해소 |
| 비주얼 제작 | 디자이너 | 소재 제작기 | VISUAL_ASSETS_SPEC | 실제 비주얼 파일 |

> 로직: 소재는 **얕은 데서 깊은 데로** — One-Pager (흥미 유발) → 10 페이지 (긴급성 확립) → 20/30 페이지 (심층 설득) → FAQ (우려 해소). 각각 동일한 다음 단계를 가리킴: 「10 문항 설문 작성, `01_Assessment` 진단 진입」.

## 5. 파일 설명

### `SALES_VALUE_PROPOSITION.md`

역할별 가치 제안과 영업 스크립트. CEO / COO / CIO / IT / HR / 부서장 각자의 페인 포인트에 대응하는 가치 제안, 30 초 / 3 분 / 10 분 3 가지 길이의 영업 스크립트와 일반적 반론 포함.

### `SALES_FAQ.md`

20 문항 대외 영업 FAQ. 고객이 가장 많이 묻는 질문 (Big-4 와의 차이, 견적 구조, 타임라인, 커스터마이징, 클라우드/온프레, 보안, ROI, 지적 재산권, AI 모델 선택, 실패 처리…), 각 2-4 문장의 표준 답변, zh+en 병기.

### `ONE_PAGER_OUTLINE.md`

One-Pager PDF / 인쇄물의 컨텐츠 스크립트와 레이아웃 브리프 (약 600 자). Hero 태그라인, 3 대 페인 포인트, 3 단계 플로우, L1-L5 지도, 검증 가능 납품물, 3 산업 적용, ROI proof point, CTA, 디자이너용 레이아웃/배색/폰트 제안 포함.

### `EXECUTIVE_DECK_OUTLINE.md`

10 페이지 C-Level 판 덱 아웃라인. 대상 CEO/COO/CIO/CTO. 각 페이지: 제목, 핵심 메시지, 내용, 비주얼 제안, 스피커 노트. 비전, 긴급성, 방법론, L1-L5, 스토리 아크, ROI, 거버넌스, 경쟁 차별화, 견적, 다음 단계 포함.

### `STANDARD_SALES_DECK_OUTLINE.md`

20 페이지 표준 영업 판 덱 아웃라인. 대상 부서장, PM, IT Lead, HR, AI Champion. 10 페이지 판보다 넓음, 각 레벨 강의 teaser, PoC 시나리오, Stage Gate, 납품 패키지, 3 단계 시작 포함.

### `CONSULTING_METHODOLOGY_DECK_OUTLINE.md`

30 페이지 컨설팅 방법론 판 덱 아웃라인. 대상 CIO, 전략장, 대형 기업 디지털 트랜스포메이션 오피스, 학술 동료. 방법론 깊이 판 — 8 단계, L1-L5 단계 연결, 채점 모델, 케이스 발췌, 거버넌스, 학술 뿌리 (Prof. Michael Rosemann, QUT).

### `VISUAL_ASSETS_SPEC.md`

3 개 대외 비주얼 소재의 ASCII 사양과 디자이너 브리프: 3 단계 서비스 플로우 다이어그램, L1-L5 성숙도 지도, 검증 가능 납품물 리스트 비주얼. 각각 ASCII 사양, 레이아웃 대안, 배색/폰트/크기/접근성 요구사항 포함.

### `*_EN.md`

위 파일들의 영어 sibling 버전.

## 7. 일반적 사용 시나리오

- **Lead 개발**: 영업이 `ONE_PAGER_OUTLINE.md` 로 만든 One-Pager DM 발송.
- **첫 회의**: `EXECUTIVE_DECK_OUTLINE.md` 로 C-Level 에 15-20 분 브리핑, 마지막에 「10 문항 설문 작성」 으로 유도.
- **심층 제안**: 대상별로 20 페이지 (부서장) 또는 30 페이지 (CIO/전략장) 선택.
- **고객이 「너무 비싸 / 우리 이미 ChatGPT 있어 / 데이터 안전한가」 말함**: `SALES_FAQ.md` 의 표준 답변 펴기.
- **비주얼 제작**: `VISUAL_ASSETS_SPEC.md` 를 디자이너에게 줘서 실제 파일 제작.
