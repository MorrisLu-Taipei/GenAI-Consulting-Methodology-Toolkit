## Part C — Eight-Stage Consulting Structure (Slides 15-22)

**15. Eight-stage overview** — As-Is → Reference Model → Best Practice → Gap → Problem → Phased Goals → To-Be → Implementation
**16. Stage 1 As-Is Snapshot** — interview question bank, AI Usage Inventory, Systems Inventory, As-Is Process Map
**17. Stage 2 Reference Model Alignment** — APQC PCF / SCOR / Tiger AI L1-L5 dual coordinates, Standard Capability Gap Checklist, Operating Structure Completeness radar
**18. Stage 3 Best Practice Integration** — Excellence Capability Profile, benchmark cases (manufacturing / hospital / retail / finance / gov)
**19. Stage 4 Gap Analysis** — Missing/Broken/Redundant, Impact×Effort (**not a risk assessment**)
**20. Stage 5 Problem Definition** — 80/20 convergence + 5 Whys + Executive Problem Statement
**21. Stage 6 Benchmarking & Phased Goals** — Decompose ultimate benchmark into Phase 1/2/3 + Stage Gate Criteria + Organizational Absorption assessment
**22. Stage 7 To-Be Design** — Phased To-Be Operating Model + Human-AI Collaboration Architecture (HITL nodes) + Skill/Workflow/Agent Map
**22b. Stage 8 Implementation & Change** — Transformation Roadmap + Change Management Playbook + Value Tracking Matrix (5 dimensions) + Risk Register + Audit + Ethics

## Part C2 — Stage 2 Tool Depth (Slides 22c-22e) — Why This Is a Reference Model

## Part C3 — Stage 3 Tool Depth (Slides 22f-22g) — Best Practice Is Material; Ideal Practice Is the Target

**22f. Tool 3.5 Cases-as-Benchmarks**: **Maturity ≡ Benchmarking** — 9 mandatory case fields (industry/scale/start L/end L/duration/investment/Wins/Failures/applicability), client self-computes gap in 30 min

**22g. Tool 3.6 Client Ideal Practice Co-Creation Workshop**: 6 steps (material display → independent voting → collective consensus → reality check → definition table → 3-party signature); **once client signs, they cannot deny their own picked targets** — the engagement anchor
