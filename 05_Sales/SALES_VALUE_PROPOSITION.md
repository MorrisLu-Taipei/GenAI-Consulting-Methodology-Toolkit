## 3. 一句話定位

我們不是只教 AI 工具，而是用問卷診斷企業目前 AI 成熟度，再依 L1-L5 配課程，課中產出可驗收資產，最後用八階段顧問方法產出 AI 轉型診斷報告與落地 Roadmap。

## 3.5 客戶上課前的未來想像

銷售與課程開場要先講未來，不要先講工具。

可以這樣說：

> 我們先不要談工具，先想像三個月後你的公司怎麼工作。每位員工都有自己的 AI 工作區，主管看得到權限與使用邊界；高手的方法被整理成 Skill，新人也能用同一套標準；客訴、報表、CRM 更新、ERP 查詢不再靠人工複製貼上，而是由 n8n 串成流程；Hermes Agent 每天產生 briefing、追蹤清單與需要人工批准的事項；當公司要做新產品、品質改善或客戶經營時，多個 Agent 可以像團隊一樣協作。L1-L5 課程就是把公司一步一步帶到這個狀態。

客戶聽完後，要能對應到以下能力：

| 等級 | 客戶聽得懂的未來能力 | 對應課程 |
| --- | --- | --- |
| L1 | 員工安全使用 AI，每人有自己的帳號與聊天區 | OpenWebUI 企業啟用 |
| L2 | 高手方法變成部門 Skill，新人能快速上手 | Antigravity / Claude Code / Codex Skill 化 |
| L3 | AI 串接系統，流程開始自動跑 | n8n Workflow AI |
| L4 | Agent 自動讀資料、產 briefing、留 evidence、等人批准 | Hermes Agent |
| L5 | 多個 Agent 形成跨部門 AI Team | ClawTeam |

這段故事應該放在問卷與課程介紹之前，因為客戶先看懂未來，才會願意回答成熟度問卷，也才知道為什麼課程要分層、為什麼每一層都要有 deliverables 與 Stage Gate。

## 3.6 采購 AI，不是買系統（L4-L5 的關鍵 reframe）

到了 L4-L5，老闆要換的不是工具，是**心態**。

很多老闆用「買系統」的方式看 AI：比功能、比授權價、算軟體 ROI。但 L4-L5 的 AI 不是軟體 —— 它會推理、會自主、會用工具，功能上做的就是一個知識工作者在做的事。所以對的心態是**「采購 AI」**：像招募、上工、管理、治理一個人才。

銷售對話可以這樣帶：

> 您現在是在「買一套 AI 軟體」，還是在「采購一位 AI 人力」？如果是後者，您該問的不是「它有什麼功能」，而是「它幫我接走多少工作量、我要怎麼管它、誰來治理它」。

幾個要點（對 CEO / 高層特別有效）：

- **ROI 不是不算，是換把尺**：別用「買軟體」的尺（功能、座位、年費），用「聘人力」的尺（接走多少工作量、放大多少產能）。
- **L1-L3 你本來就是買工具 / 設備 / 系統** —— 那是合理且可驗證的；「采購 AI」這個 reframe 只用在 L4-L5。
- **它是管理姿態的類比，不是說「AI 就是人」** —— 正因為 AI ≠ 人，它才需要人建立的治理框架，而治理正是我們的服務核心。

## 4. 角色價值主張

### 4.1 CEO

CEO 關心的是：AI 投資到底能不能變成經營成果。

我們提供：

- 一張看得懂的 AI 成熟度地圖。
- 各部門目前在哪一級的客觀判斷。
- 哪些 AI 場景最有 ROI。
- 三階段 Roadmap。
- 可以追蹤的 KPI 與 Stage Gate。

對 CEO 的說法：

> 這套方案會讓您知道公司現在 AI 到哪裡、下一步該投資哪裡，以及哪些場景最有機會在 90 天內看到成果。

### 4.2 COO

COO 關心的是：流程是否更快、更穩、更少重工。

我們提供：

- As-Is 流程盤點。
- 人工重工點整理。
- Gmail、Sheets、Notion、CRM、API、ERP 串接場景。
- n8n Workflow PoC。
- 跨部門流程改善建議。

對 COO 的說法：

> 我們會把 AI 從個人聊天工具，帶進實際營運流程，優先處理最重複、最耗時、最容易漏掉的工作。

### 4.3 CIO / IT

CIO / IT 關心的是：資料安全、權限、維運、稽核與系統整合。

我們提供：

- 雲 AI / Hybrid / 全地端部署判斷。
- Credential、API、權限、Log 盤點。
- Hermes Agent 可用工具、Wiki 記憶、索引、Log 與權限邊界。
- 人工審核與例外處理機制。
- 治理與維運建議。

對 CIO / IT 的說法：

> 這不是讓各部門自己亂串工具，而是把 AI 使用、流程自動化與 Agent 權限納入可治理的架構。

### 4.4 HR / 訓練單位

HR 關心的是：課程上完後是否真的改變工作方式。

我們提供：

- 課前成熟度診斷。
- 依部門分組的課程設計。
- 課中產出 Prompt、Skill、Workflow、Agent 任務。
- 課後能力盤點。
- 部門 Skill Library 與訓練追蹤。

對 HR 的說法：

> 這不是一次性 AI 課程，而是把訓練成果變成部門可複用的工作資產。

### 4.5 部門主管

部門主管關心的是：團隊能不能少做重複工作、品質更穩、新人更快上手。

我們提供：

- 部門高頻工作盤點。
- Skill 化。
- Workflow PoC。
- Hermes Agent 任務卡、briefing 與追蹤清單。
- 部門 KPI 建議。

對部門主管的說法：

> 我們會先找出你們部門最常做、最耗時、最容易出錯的工作，轉成 Skill、Workflow 或 Agent 任務。

## 5. 30 秒銷售話術

多數企業現在不是完全沒用 AI，而是大家各自用，成果散落、流程沒改、ROI 不清楚。我們會先用問卷診斷企業 AI 成熟度，再依 L1-L5 安排課程比例，課中做出 Prompt、Skill、Workflow 與 Hermes Agent evidence，最後用八階段顧問方法產出 AI 轉型診斷報告，讓企業知道現在在哪裡、下一步做什麼、哪些流程可以先自動化。

## 6. 3 分鐘銷售話術

企業導入 AI 最常見的狀態，是員工已經開始用 ChatGPT、Claude 或 Gemini，但公司層面看不到組織能力。有人會寫 Prompt，有人會整理資料，有人想串 CRM 或 ERP，但缺少共同架構。

我們把 AI 成熟度分成五級。L1 是 OpenWebUI 的 Chat AI，讓員工安全開始使用 AI。L2 是 Skill AI，用 Antigravity、Claude Code、Codex 把個人經驗變成部門 Skill。L3 是 n8n Workflow AI，串接 Gmail、Sheets、Notion、CRM、API、ERP，讓 AI 進入流程。L4 是 Hermes Agent，讓 AI 有 Wiki 記憶、能吸收文件、呼叫工具、產生 briefing，並用 Log 與 evidence 證明結果。L5 是 ClawTeam，也就是 AI Agent Team 協作平台，支援跨部門任務。

我們的服務分三段。第一段用問卷快速診斷企業現在在哪一級。第二段依診斷結果配置課程比例，讓客戶在課程中產出 Prompt、Skill、Workflow、Hermes Agent PoC 與 Stage Gate evidence。第三段用八階段顧問方法整理現況、差距、核心問題、To-Be 藍圖、Roadmap 與 ROI。

所以客戶買到的不是一堂 AI 課，而是一條從 AI 學習到企業轉型落地的路。

## 7. 10 分鐘簡報故事線

### Slide 1：企業不是沒有 AI，而是 AI 很零散

員工各自用工具，主管看不到 ROI，IT 擔心資安，部門不知道如何落地。

### Slide 2：AI 轉型是成熟度演進

從 L1 Controlled AI Access 到 L5 Multi-Agent Organization，每一級都有不同管理重點與治理能力。

### Slide 3：L1-L5 模型

- L1 OpenWebUI：個人效率。
- L2 Knowledge Codification：部門能力。
- L3 n8n：流程自動化。
- L4 Hermes Agent：具備 Wiki 記憶、briefing、工具呼叫與 evidence 的知識型 Agent。
- L5 ClawTeam：AI Agent Team 協作平台。

### Slide 4：先診斷，再上課

用 10 題或 25 題問卷判斷成熟度、公司屬性、部署模式與優先場景。

### Slide 5：課程不是固定菜單

依成熟度、產業、部署模式與系統條件配置 L1-L5 比例。

### Slide 6：讓 AI 進入系統與流程

用 n8n 串接 Gmail、Sheets、Notion、CRM、API、ERP，讓 AI 不只是回答問題，而是幫公司做事。

### Slide 7：從 Workflow 到 Agent

Hermes Agent 可以呼叫 Skill 與 Workflow，把文件、SOP、FAQ、研究資料與流程輸出整理成可追溯的 Wiki 記憶，並定期產生 briefing、追蹤清單與人工審核提醒。

### Slide 8：從 Agent 到 AI Team

ClawTeam 讓市場、產品、行銷、客服、財務等 Agent 協作完成企業級任務。

### Slide 9：課後產出顧問診斷報告

用八階段方法整理現況、差距、核心問題、To-Be 設計、Roadmap、ROI 與治理。

### Slide 10：90 天落地路線

第一階段 Skill 化，第二階段 Workflow PoC，第三階段 Agent 與治理擴大。

## 8. 常見反對意見

### Q1：我們已經有人在用 ChatGPT，還需要這個嗎？

需要。個人會用 AI 不等於公司完成 AI 轉型。差別在於能否沉澱成 Skill、串成 Workflow、納入治理、追蹤 ROI。

### Q2：我們資料不能上雲，還能做嗎？

可以。方案會先判斷雲 AI、Hybrid、全地端。高敏感資料可留在地端，低敏感任務可視政策使用雲端。

### Q3：課程上完會不會就結束？

不會。課程只是能力建置，課後會進入顧問診斷報告，整理可落地的 Roadmap 與 PoC。

### Q4：IT 會不會很難維護？

我們會把 Credential、權限、Log、人工審核、Owner 與維運責任列入設計，不會讓各部門各自亂接。

### Q5：怎麼看 ROI？

優先用時間節省、錯誤率、回應速度、流程處理量、知識沉澱、新人上手時間與 PoC 轉上線率追蹤。

### Q6：AI 工具我們自己買就好，為什麼要找顧問？

到 L1-L3，你確實是在買工具、買設備 —— 而且光是把 OpenWebUI、Skill、n8n 串 CRM / ERP 真的跑起來，技術門檻就不低。到了 L4-L5，情況更不一樣：那不是「買一套軟體」，是「采購一位會自主工作的 AI 人力」—— 它需要的是職責定義、治理框架、HITL 與究責，那是工具買不到、要顧問陪你建的（見 §3.6）。
