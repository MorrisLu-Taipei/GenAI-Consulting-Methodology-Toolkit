## 3. 무엇을 인용했나

| 범위 | 처리 |
| --- | --- |
| **케이스 이름과 13 카테고리** | 인덱스 항목으로 인용 |
| **L1-L5 매핑, 코스 매핑, 적용 시나리오 노트** | 본 방법론의 오리지널 작업 — 우리 자신의 매핑 |
| **앱의 소스 코드** | 재현하지 않음, 포크하지 않음; 학습자는 구축하고 싶다면 상류에서 클론 |
| **「인덱스」 아티팩트 자체** | 본 방법론에 오리지널; 본 repo 의 Apache 2.0 하 |

> 즉: 케이스 리스트는 awesome-llm-apps 에서; 그 케이스를 L1-L5 와 우리 코스에 매핑하는 것은 우리 작업.

## 4. 라이선스 요약

awesome-llm-apps 는 **Apache License 2.0** 하 — 자유로운 포크, 커스터마이즈, 상업적 사용을 허용. 본 방법론은 소스 코드를 **재배포하지 않음** — `LLM_APPS_CASE_INDEX.md` 에서 케이스 이름과 카테고리를 인덱스로만 인용. 실제로 앱을 실행하고 싶은 학습자는 상류 repo 에서 얻고, 그 Apache 2.0 조건 (저작권 및 라이선스 통지 보존) 을 준수해야 함.

## 6. 면책 조항

본 방법론에서 awesome-llm-apps 케이스의 L1-L5 와 코스로의 매핑은 방법론 저자 (Morris Lu / Tiger AI 虎智科技) 의 해석을 나타내며, Shubham Saboo 또는 커뮤니티 기여자의 공식 입장을 나타내지 않음. 앱은 대부분 데모 / 스타터 템플릿; 기업 채택은 본 방법론의 거버넌스 메커니즘 (Human Gate, Evidence, Stage Gate, 보안 검토) 을 적용해야 함. 상류 케이스 리스트가 새 버전에서 변경되면, [상류 리포지토리](https://github.com/Shubhamsaboo/awesome-llm-apps) 를 참조.
