# 短摘要 / Short Abstract — ZH（投稿用）

傳統電子書與專業知識型書籍延續印刷邏輯：作者以長週期撰稿，產出靜態文本，讀者以被動閱讀為主要消費方式。本文以設計科學研究方法探討一種新範式 —— **AI 原生電子書生產（AI-Native eBook production）**：電子書成為由多 IDE 協作環境（Claude Code、Cursor、Antigravity、Codex）、Git 版本控制、持久識別符（DOI）、宣告式 workflow 與讀者—IDE 互動共構的**可查詢、可審查、可重現**知識製品。我們以《GenAI Consulting Methodology Toolkit》（Apache 2.0；354 份文件；7 種語言；橫跨三個 IDE 家族的 22 個專門化 AI-IDE workflow）作為此範式的實作案例，並以顧問方法論作為**最嚴苛情境的比較標的**——因其要求高結構完整性與跨文件一致性。製品展現四項過去單一工具寫作未能達成的特性：(1) 強制執行的跨語一致性；(2) 多引擎對抗式審查（如 `/devil-pair-debate`、`/red-team-review`）；(3) 讀者可查詢執行——生產該製品的 IDE 可透過 `/socratic-challenge`、`/deep-synthesize` 被讀者反向詢問；(4) 透過 Git 歷史與宣告式 workflow 達成的**可加密重現衍生鏈**。我們依 Hevner 設計科學七大準則進行評估、與傳統方法論發展循環（McKinsey 7-Step、Rosemann BPM）量化對比，並提出將 Git / GitHub / Zenodo 機械化為 **PDCA 出版循環**的觀點。最後討論此範式對電子書生產、方法論工程、HCI 中「IDE 即媒介」命題、以及 LLM 時代「active」學術出版的意涵。

**關鍵字：** 設計科學研究；AI IDE；AI 原生電子書；電子書生產；專業知識製品；多代理協作；可執行文件；可重現製品；literate programming；顧問框架
