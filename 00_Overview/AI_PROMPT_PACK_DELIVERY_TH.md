# เล่มที่ 2 AI Prompt Pack (การส่งมอบงานที่ปรึกษา)

> 🌐 ภาษา: [繁體中文](AI_PROMPT_PACK_DELIVERY.md) ｜ [English](AI_PROMPT_PACK_DELIVERY_EN.md) ｜ **ไทย**
> เป็นคู่ของ [`11_AI_Prompt_Pack.md`](https://github.com/MorrisLu-Taipei/consultant-thinking-framework/blob/main/11_AI_Prompt_Pack.md) ในเล่มที่ 1 แพ็กของเล่มที่ 1 ฝึก *การตัดสิน* ส่วนแพ็กนี้ผลักดันการตัดสินนั้นผ่านทุกขั้นของ *การส่งมอบ*
> วินัยร่วม: **แยกสัญญาณออกจากการอนุมาน ห้ามกุขึ้นเอง ทำเครื่องหมายสิ่งที่ยังไม่ได้ยืนยันว่า "รอยืนยัน" ประมวลผลข้อมูลลูกค้าด้วย AI ที่ได้รับอนุมัติ / เอนด์พอยต์ส่วนตัวเท่านั้น**

---

## วิธีใช้
วางข้อมูลลูกค้า / บันทึกการสัมภาษณ์ลงใน `{{ }}` ของแต่ละพรอมป์ต์ หรือเรียกใช้ผ่าน skill `consulting-delivery-coach` ทุกพรอมป์ต์จะขอให้ AI **อ้างอิงแหล่งที่มาของ evidence** และ **ระบุพื้นฐานของระดับ L**

---

### 1. โปรไฟล์ลูกค้า → ระดับ L เบื้องต้น
```
คุณคือที่ปรึกษาวินิจฉัยการปฏิรูปด้วย AI จากภูมิหลังลูกค้าด้านล่าง จงให้คำตัดสินวุฒิภาวะ L1-L5 เบื้องต้น
กฎ: ผูกทุกคำตัดสินเข้ากับ evidence ที่เฉพาะเจาะจงในภูมิหลัง มิติที่มีหลักฐานไม่เพียงพอให้ทำเครื่องหมายว่า "รอวินิจฉัย" — ห้ามเดา
ผลลัพธ์: ระดับ L เบื้องต้น + คำถามติดตามที่สำคัญที่สุด 3 ข้อ
ภูมิหลังลูกค้า: {{วาง}}
```

### 2. บันทึกการสัมภาษณ์ → สรุปการวินิจฉัย Phase A
```
จงเปลี่ยนถอดเทปการสัมภาษณ์ด้านล่างให้เป็นสรุปการวินิจฉัย Phase A
โครงสร้าง: (1) คำพูดตรงตัวของลูกค้า (ข้อเท็จจริง) (2) ความเจ็บปวดที่แท้จริง (ผิว → ราก) (3) job-to-be-done ปัจจุบัน (4) เส้นทแยงมุม (diagonal) (จุดบอดของคู่แข่ง × ความต้องการอนาคตของลูกค้า × จุดเข้าของเรา) (5) ช่องว่างของ evidence (ทำเครื่องหมาย "รอยืนยัน")
อย่าเพิ่มสิ่งที่ไม่มีอยู่ในถอดเทป
ถอดเทป: {{วาง}}
```

### 3. ผลการวินิจฉัย → ข้อเสนอแนะการออกแบบหลักสูตร
```
จากการวินิจฉัยระดับ L ด้านล่าง จงเสนอแนะเส้นทางหลักสูตร L1-L5
กฎ: ตัดสินก่อนว่าลูกค้าขาด "ความสามารถ / กระบวนการ / ธรรมาภิบาล / เครื่องมือ" อย่างใด แล้วจึงออกแบบหลักสูตร อธิบายเหตุผลของลำดับ
ผลลัพธ์: เส้นทางหลักสูตร + ผลผลิตเป้าหมายต่อขั้น + ความเสี่ยง
การวินิจฉัย: {{วาง}}
```

### 4. ผลการวินิจฉัย → โครงร่างรายงานที่ปรึกษา
```
จงเปลี่ยนการวินิจฉัยด้านล่างให้เป็นโครงร่างรายงานที่ปรึกษาแปดขั้น
ต้องมี: Executive Problem Statement (เขียนจากเส้นทแยงมุม (diagonal)), Roadmap, Risk Register, Stage Gate Criteria
อ้างอิงแหล่งที่มาของ evidence สำหรับทุกข้อสรุป อย่าเขียนสิ่งใดเป็นข้อสรุปโดยไม่มี evidence
การวินิจฉัย: {{วาง}}
```

### 5. Roadmap → เช็กลิสต์การส่งมอบ
```
จงเปลี่ยน Roadmap ด้านล่างให้เป็นเช็กลิสต์การส่งมอบที่พร้อมตรวจรับ
กฎ: แต่ละรายการต้องมี "เกณฑ์การตรวจรับ" (อะไรนับว่าส่งมอบสำเร็จ) + "สิ่งนี้ส่งมอบความสามารถหรือเครื่องมือ" รายการที่ส่งมอบเพียงเครื่องมือให้ทำเครื่องหมายเตือน
Roadmap: {{วาง}}
```

### 6. สถานะโครงการ → ความเสี่ยงและขั้นถัดไป
```
คุณคือ reviewer ของโครงการ จากสถานะโครงการด้านล่าง จงระบุความเสี่ยงและขั้นถัดไป
จุดเน้น: การขยายขอบเขต (scope creep), ควรเปิด Stage Gate หรือไม่, ความเสี่ยงของงวดสุดท้าย, โครงสร้างของลูกค้าดีขึ้นจริงหรือไม่
ผลลัพธ์: ความเสี่ยง Top 3 + ขั้นถัดไปที่แนะนำ + ควรเข้าสู่ Phase A/B/C ถัดไปหรือไม่
สถานะ: {{วาง}}
```

- ที่ปรึกษาที่เป็นมนุษย์ต้องตรวจทานผลลัพธ์ของ AI ทั้งหมด **สิ่งที่ทำเครื่องหมายว่า "รอยืนยัน" ต้องไม่ถูกส่งมอบเป็นข้อสรุป**
