# 하권 AI Prompt Pack (컨설팅 프로젝트 도입·딜리버리)

> 🌐 언어: [繁體中文](AI_PROMPT_PACK_DELIVERY.md) ｜ [English](AI_PROMPT_PACK_DELIVERY_EN.md) ｜ **한국어**
> 상권의 [`11_AI_Prompt_Pack.md`](https://github.com/MorrisLu-Taipei/consultant-thinking-framework/blob/main/11_AI_Prompt_Pack.md)에 대응합니다. 상권 Pack은 *판단*을 단련하고, 본 Pack은 그 판단을 *딜리버리*의 각 단계로 밀고 나갑니다.
> 공통 규율: **신호와 추론을 구분한다, 절대 지어내지 않는다, 검증되지 않은 것은 "확인 필요"로 표기한다, 고객 데이터는 승인된 AI / 프라이빗 엔드포인트로만 처리한다.**

---

## 사용 방법
고객 데이터 / 인터뷰 기록을 각 프롬프트의 `{{ }}`에 붙여 넣거나 `consulting-delivery-coach` skill로 트리거하세요. 모든 프롬프트는 AI에게 **evidence 출처를 인용할 것**과 **L 레벨의 근거를 명시할 것**을 요구합니다.

---

### 1. 고객 프로필 → 예비 L 레벨
```
당신은 AI 전환 진단 컨설턴트입니다. 아래 고객 배경을 바탕으로 예비 L1-L5 성숙도 판단을 제시하세요.
규칙: 각 판단은 배경 속의 구체적인 evidence에 연결할 것. 증거가 불충분한 차원은 "진단 필요"로 표기하고 추측하지 말 것.
출력: 예비 L 레벨 + 가장 보완해야 할 후속 질문 3가지.
고객 배경: {{붙여넣기}}
```

### 2. 인터뷰 기록 → Phase A 진단 요약
```
아래 인터뷰 녹취록을 Phase A 진단 요약으로 정리하세요.
구조: (1) 고객의 원문 그대로(사실) (2) 진짜 통점(표층 → 근본 원인) (3) 현재의 job-to-be-done (4) 대각선(diagonal)(경쟁사의 맹점 × 고객의 미래 니즈 × 우리의 진입점) (5) evidence 공백("확인 필요"로 표기).
녹취록에 없는 정보는 추가하지 말 것.
녹취록: {{붙여넣기}}
```

### 3. 진단 결과 → 커리큘럼 설계 제안
```
아래 L 레벨 진단을 바탕으로 L1-L5 커리큘럼 경로를 제안하세요.
규칙: 먼저 고객에게 부족한 것이 "역량 / 프로세스 / 거버넌스 / 도구" 중 무엇인지 판단한 뒤 커리큘럼을 설계할 것. 순서의 이유를 설명할 것.
출력: 커리큘럼 경로 + 단계별 목표 산출물 + 리스크.
진단: {{붙여넣기}}
```

### 4. 진단 결과 → 컨설팅 보고서 개요
```
아래 진단을 8단계 컨설팅 보고서 개요로 변환하세요.
필수 포함: Executive Problem Statement(그 대각선(diagonal)에서 작성), Roadmap, Risk Register, Stage Gate Criteria.
각 결론에 evidence 출처를 명시할 것. 증거가 없는 것을 결론으로 쓰지 말 것.
진단: {{붙여넣기}}
```

### 5. Roadmap → 딜리버리 체크리스트
```
아래 Roadmap을 검수 가능한 딜리버리 체크리스트로 변환하세요.
규칙: 각 항목에 "검수 기준"(무엇을 딜리버리 성공으로 보는가) + "이것이 역량을 전달하는가 도구를 전달하는가"를 기재할 것. 도구만 전달하는 항목에는 경고를 표시할 것.
Roadmap: {{붙여넣기}}
```

### 6. 프로젝트 상태 → 리스크와 다음 단계
```
당신은 프로젝트 reviewer입니다. 아래 프로젝트 상태를 바탕으로 리스크와 다음 단계를 나열하세요.
중점: 범위 확산(scope creep), Stage Gate를 발동해야 하는지, 잔금 리스크, 고객 구조가 정말 개선되었는지.
출력: Top 3 리스크 + 권장 다음 단계 + 다음 Phase A/B/C로 진입할지 여부.
상태: {{붙여넣기}}
```

## 안전성과 정직성에 관한 주의
- 고객 데이터는 **회사가 승인한 AI 또는 프라이빗 엔드포인트**로만 처리할 것. 공개 서비스에 붙여 넣지 말 것.
- AI의 출력은 모두 인간 컨설턴트가 복심할 것. **"확인 필요"로 표기된 것은 결론으로 납품해서는 안 된다**.
- 대외 사례는 비식별화할 것("Company ABC" / "City X"). 본 Pack은 도구이며, 법률 / 재무 / 보안의 최종 의견이 아닙니다.
