# 下集 AI Prompt Pack(顧問專案導入)

> 🌐 語言:繁體中文 ｜ [English](AI_PROMPT_PACK_DELIVERY_EN.md)
> 對應上集的 [`11_AI_Prompt_Pack.md`](https://github.com/MorrisLu-Taipei/consultant-thinking-framework/blob/main/11_AI_Prompt_Pack.md)。上集 Pack 練「判斷」;本 Pack 把判斷推進「專案交付」每一階段。
> 共同紀律:**區分訊號與推論、不得虛構、未證實標為「待確認」、客戶資料只用核准 AI/私有端點**。

---

## 怎麼用
把客戶資料 / 訪談紀錄貼進對應 prompt 的 `{{ }}`,或直接用 `consulting-delivery-coach` skill 觸發。每個 prompt 都會要求 AI **標註 evidence 來源**與**標出 L 級依據**。

---

### 1. 客戶資料 → 初步 L 級判斷
```
你是 AI 轉型診斷顧問。根據以下客戶背景,給出初步 L1-L5 成熟度判斷。
規則:每個判斷要指到背景中的具體證據;證據不足的維度標「待診斷」,不要猜。
輸出:初步 L 級 + 3 個最該補問的問題。
客戶背景:{{貼上}}
```

### 2. 訪談紀錄 → Phase A 診斷摘要
```
把以下訪談逐字稿整理成 Phase A 診斷摘要。
結構:① 客戶原話(事實)② 真正痛點(表層→根因)③ 現在的任務 ④ 對角線(對手盲點×客戶未來需求×我們切入)⑤ 證據缺口(標待確認)。
不要加訪談中沒有的資訊。
逐字稿:{{貼上}}
```

### 3. 診斷結果 → 課程配置建議
```
根據以下 L 級診斷,建議 L1-L5 課程路線。
規則:先判斷客戶缺的是「能力 / 流程 / 治理 / 工具」哪一類,再配課;說明為何這樣排序。
輸出:課程路線 + 每階段目標產出 + 風險。
診斷:{{貼上}}
```

### 4. 診斷結果 → 顧問報告大綱
```
把以下診斷轉成八階段顧問報告大綱。
必含:Executive Problem Statement(用那條對角線寫)、Roadmap、Risk Register、Stage Gate Criteria。
每個結論標 evidence 來源;沒有證據的不要寫成結論。
診斷:{{貼上}}
```

### 5. Roadmap → Delivery checklist
```
把以下 Roadmap 轉成可驗收的交付清單。
規則:每一項要寫「驗收標準」(怎樣算交付成功)+「交付的是能力還是工具」。只交工具的項目要標警示。
Roadmap:{{貼上}}
```

### 6. 專案狀態 → 風險與下一步
```
你是專案 reviewer。根據以下專案狀態,列出風險與下一步。
重點:範圍蔓延、Gate 是否該觸發、尾款風險、客戶結構是否真的變好。
輸出:Top 3 風險 + 建議下一步 + 是否該進下一 Phase。
狀態:{{貼上}}
```

---

## 安全與誠實提醒
- 客戶資料只用**公司核准 AI 或私有端點**;不貼公開服務。
- AI 產出一律由人類顧問複核;**標「待確認」的不得當成結論交付**。
- 對外案例去識別化(ABC公司 / 城市X)。本 Pack 是工具,不是法律/財務/資安最終意見。
