## 序章:這本書要解決什麼
上集把你練成「能判斷」的顧問;下集教你把那份判斷,變成**可交付、可驗收、可續約**的專案。核心三紀律貫穿全書:
1. **交的是能力,不是工具。** 2. **每個結論要有 evidence。** 3. **Phase 之間要有 Gate。**

🤖 **練習 0**:用一句話寫下你手上案子的「客戶真正痛點」,以及一條對角線(對手盲點 × 客戶未來需求 × 我們切入)。寫不出來→先回上集。

---

## 第1章:先看全貌——四張別搞混的圖
跑案前先分清四張圖(權威定義見 `CANONICAL_TERMS_AND_MAPS.md`):
- **服務流程(鳥瞰)**:診斷 → 建置 → 交付。
- **顧問八階段(方法論本體)**:As-Is → Reference Model → Ideal Practice → Gap → Problem Definition → Phased Goals → To-Be → Implementation。
- **合約 Phase(賣法/Gate)**:A 診斷 2 週 / B 策略 4 週 / C 落地 24 個月。
- **接案生命週期(經營)**:Sales → Delivery → Support(`../06_Delivery/ENGAGEMENT_LIFECYCLE.md`)。

🤖 **練習 1**:把你的案子目前落在「合約 Phase 哪一段、生命週期哪一步」各標一個。混在一起講的人,通常會把報價、診斷、交付攪成一團。

---

## 第2章:診斷——把訪談變判斷(Phase A,2 週)
先 10 題快篩定初步**企業 L 級**,再 25/50 題正式診斷,每個判斷指到 evidence。企業五級正式名(`../01_Assessment/AI_MATURITY_SCORING_MODEL.md`):
L1 Controlled AI Access · L2 Knowledge Codification · L3 Workflow Automation · L4 Autonomous Agent · L5 Multi-Agent Organization。
Phase A 交付物=**中期評估報告(客戶簽收)**,然後 **Gate A** 決定要不要進 B。

🤖 **練習 2**(skill 模式①②):把一段訪談紀錄丟給 `consulting-delivery-coach`,要它輸出「初步 L 級 + 證據 + 3 個最該補問的問題」。沒證據的維度要它標「待診斷」,不准猜。

---

## 第3章:配課與報告——把判斷變方案(Phase B,4 週)
先判客戶缺的是**能力 / 流程 / 治理 / 工具**哪一類,再配 L1-L5 課程路線(`../02_Course_Design/`)。報告用八階段大綱,必含:Executive Problem Statement(**用那條對角線寫**)、Roadmap、Risk Register、Stage Gate Criteria(`../03_Consulting_Report/CONSULTING_REPORT_TEMPLATE.md`)。交付物=**完整顧問報告**,**Gate B** 決定進不進 C。

🤖 **練習 3**(skill 模式③④):把診斷結果丟給教練,要它先說「客戶缺的是能力/流程/治理/工具哪類、為什麼」,再生報告大綱;每個結論標 evidence,沒證據的不准寫成結論。

---

## 第4章:交付驗收——交能力不交工具(Phase C 起,24 個月)
把 Roadmap 變成**可驗收清單**:每項寫「驗收標準(怎樣算成功)」+「交的是能力還是工具」,只交工具的標警示(`../06_Delivery/`)。落地後**每季回核 Stage 2 雷達**——結構真的變好了嗎?這就是 **Gate C** 與續約的理由。

🤖 **練習 4**(skill 模式⑤):把你的 Roadmap 丟給教練轉成交付驗收清單,挑出「其實只交了工具、沒交能力」的項目。

---

## 第5章:把案子當生意跑——風險與續約
用三階段合約模型控範圍:**先賣 Phase A,用 Gate 決定往下,別一開始賣完整 24 個月**。常見死法:範圍蔓延、沒 Gate、報告沒 evidence、把工具當轉型、收不到尾款(`BEGINNER_DELIVERY_GUIDE.md` §五大錯)。

🤖 **練習 5**(skill 模式⑥):把專案現況丟給教練,要 Top 3 風險 + 下一步 + 是否該觸發 Gate。
