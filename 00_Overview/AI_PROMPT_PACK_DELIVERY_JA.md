# 下巻 AI Prompt Pack(コンサルティング案件の導入・デリバリー)

> 🌐 言語：[繁體中文](AI_PROMPT_PACK_DELIVERY.md) ｜ [English](AI_PROMPT_PACK_DELIVERY_EN.md) ｜ **日本語**
> 上巻の [`11_AI_Prompt_Pack.md`](https://github.com/MorrisLu-Taipei/consultant-thinking-framework/blob/main/11_AI_Prompt_Pack.md) に対応します。上巻の Pack は*判断*を鍛え、本 Pack はその判断を*デリバリー*の各段階へと進めます。
> 共通の規律：**シグナルと推論を区別する、決して捏造しない、未確認のものは「要確認」と明記する、顧客データは承認済み AI / プライベートエンドポイントでのみ処理する。**

---

## 使い方
顧客データ / インタビュー記録を各プロンプトの `{{ }}` に貼り付けるか、`consulting-delivery-coach` skill で起動してください。各プロンプトは AI に対して **evidence の出典を明示すること** と **L レベルの根拠を示すこと** を求めます。

---

### 1. 顧客プロフィール → 暫定 L レベル
```
あなたは AI トランスフォーメーション診断コンサルタントです。以下の顧客背景から、暫定の L1-L5 成熟度判断を示してください。
ルール：各判断は背景中の具体的な evidence に結びつけること。証拠が不十分な次元は「要診断」と明記し、推測しないこと。
出力：暫定 L レベル + 最も補うべき追加質問 3 つ。
顧客背景：{{貼り付け}}
```

### 2. インタビュー記録 → Phase A 診断サマリー
```
以下のインタビュー逐語録を Phase A 診断サマリーにまとめてください。
構成：(1) 顧客の発言そのまま(事実) (2) 真の痛点(表層 → 根本原因) (3) 現在の job-to-be-done (4) 対角線(diagonal)(競合の盲点 × 顧客の将来ニーズ × 当社の切り口) (5) evidence の欠落(「要確認」と明記)。
逐語録にない情報は加えないこと。
逐語録：{{貼り付け}}
```

### 3. 診断結果 → カリキュラム設計の提案
```
以下の L レベル診断に基づき、L1-L5 のカリキュラム経路を提案してください。
ルール：まず顧客に不足しているのが「能力 / プロセス / ガバナンス / ツール」のどれかを判断し、その上でカリキュラムを設計する。順序の理由を説明すること。
出力：カリキュラム経路 + 各段階の目標成果 + リスク。
診断：{{貼り付け}}
```

### 4. 診断結果 → コンサルティングレポートのアウトライン
```
以下の診断を八段階のコンサルティングレポートのアウトラインに変換してください。
必須項目：Executive Problem Statement(その対角線(diagonal)から記述)、Roadmap、Risk Register、Stage Gate Criteria。
各結論には evidence の出典を明示すること。証拠のないものを結論として書かないこと。
診断：{{貼り付け}}
```

### 5. Roadmap → デリバリーチェックリスト
```
以下の Roadmap を検収可能なデリバリーチェックリストに変換してください。
ルール：各項目に「検収基準」(何をもってデリバリー成功とするか)+「これは能力を届けるのかツールを届けるのか」を記載すること。ツールのみを届ける項目には警告を付すこと。
Roadmap：{{貼り付け}}
```

### 6. プロジェクト状況 → リスクと次のステップ
```
あなたはプロジェクトの reviewer です。以下のプロジェクト状況から、リスクと次のステップを挙げてください。
重点：スコープクリープ、Stage Gate を発動すべきか、残金リスク、顧客の構造が本当に改善したか。
出力：Top 3 リスク + 推奨される次のステップ + 次の Phase A/B/C に進むべきか。
状況：{{貼り付け}}
```

---

## 安全性と誠実性に関する注意
- 顧客データは **会社が承認した AI またはプライベートエンドポイント** でのみ処理すること。公開サービスには貼り付けないこと。
- AI の出力はすべて人間のコンサルタントが複核すること。**「要確認」と明記されたものは結論として納品してはならない**。
- 対外事例は匿名化すること(「Company ABC」/「City X」)。本 Pack はツールであり、法務 / 財務 / セキュリティの最終的な意見ではありません。
