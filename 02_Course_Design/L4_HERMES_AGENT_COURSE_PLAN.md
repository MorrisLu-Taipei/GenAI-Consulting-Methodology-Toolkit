參考素材：知識型 Agent 架構之內部實作經驗（Hermes Agent）

## 1. L4 重新定位

L4 不只是「讓 AI 自己跑任務」。在本方法論中，L4 Hermes Agent 的定位是：

> 把 L1 Chat、L2 Skill、L3 Workflow 升級成可持續運行的知識型 Agent 作業系統。

Hermes Agent 不是單一 Prompt，也不是只會觸發 API 的機器人。它應該具備：

1. **可持續的任務記憶**：以 Markdown 知識庫作為知識真相來源。
2. **可查詢的索引能力**：以索引資料庫（可由 Markdown 重建的衍生快取）作為查詢加速層。
3. **可組合的技能**：以「1 個主控技能 + 數個專業技能」組成 Agent 能力。
4. **可驗證的工具鏈**：用 deterministic 工具完成寫入、索引、驗證、抽取、快取、刪除保護與紀錄等「帳務性」工作。
5. **可排程的工作節奏**：用夜間消化、晨間簡報、定期探勘、晚間預覽、週度知識圖整合形成固定營運節奏。
6. **可治理的人機協作**：高風險動作保留人工 Gate，所有輸出需有證據、Log、來源與審核紀錄。

因此，L4 課程的重點不是讓學員相信 Agent 很厲害，而是讓企業知道如何把 Agent 放進可控流程中，讓它每天穩定產生可驗證的工作成果。

## 2. L4 知識型 Agent 的七大設計原則

這七個原則是 L4 課程的思想核心。學員若只學工具操作而不懂這些原則，做出的 Agent 會停在「會跑的 demo」，無法變成「會長大的組織能力」。

### 2.1 白天輕、夜間重

Agent 的工作節奏應該分時：

- **白天**：只做輕量、即時的事 — 收檔案、回答問題、記錄任務。不打斷使用者。
- **夜間**：做重量、批次的事 — 消化當天收進來的資料、寫入知識庫、建立跨參照、重建知識圖。

這讓 Agent 在使用者上班時「不吵」，下班後「自己長大」。

### 2.2 知識複利閉環

知識型 Agent 的價值來自**閉環自我成長**，而不是單次處理：

```text
吃進資料 → 抽出關鍵字候選 → 人工勾選 → 加入觀察清單
   ↑                                          ↓
再吃進來  ←  進待辦佇列  ←  自動探勘外部來源  ←
```

每一次 ingest 都讓下一次 ingest 更聰明。閉環任何一環斷掉（例如忘了抽關鍵字），知識庫就停止成長。

### 2.3 人類提報永遠優先於自動探勘

任務有兩個來源，優先級不同：

- **P1 — 人類提報**：使用者主動丟進來的，**當晚必清**。
- **P2 — 自動探勘**：Agent 自己從外部找到的，排在 P1 之後。

Agent 不能因為「自己找到一堆有趣的東西」就把使用者交辦的事往後延。人的需求永遠優先。

### 2.4 寫讀同源

查詢的答案可以選擇「回寫」進知識庫。於是：

> 問問題這個行為，本身也在養大知識庫。

下一次有人問類似問題，就能撈到上次整理好的答案。讀與寫共用同一個知識源，不是兩套系統。

### 2.5 工具做帳務、LLM 做判讀，不可互換角色

明確分工：

| 角色 | 負責 | 為什麼 |
|---|---|---|
| **Deterministic 工具** | 寫檔、索引、SQL upsert、格式驗證、多格式抽取、紀錄 append | 這些要 100% 準確、可重複，不該由 LLM 來「猜」 |
| **LLM / Agent** | 閱讀、分析、判斷、跨資料推理、合成答案 | 這些需要理解與判斷，工具做不到 |

兩者角色互換就會出事 — 讓 LLM 去算雜湊或寫資料庫會出錯，讓工具去判斷「這篇論文重不重要」則做不到。

### 2.6 失敗模式驅動學習

把**真實事故**記錄下來，當成 Agent 與團隊的教材：

> 「2026-XX-XX：Agent 沒先讀現況就直接寫入，造成重複頁面、沒有 Log」→ 修正：任何寫入前一定先 orient（讀目的、結構、索引、現況）。

每個失敗模式都對應一條「不可重蹈」的規則。L4 課程要教學員建立自己的「失敗模式清單」。

### 2.7 為什麼不只用 RAG

L4 要讓學員理解「知識型 Agent」與「純 RAG」的差別：

| 方案 | 問題 |
|---|---|
| **純 RAG** | 每次查詢都從原始文件重看一遍 — 成本隨資料量線性上升，知識**不複利** |
| **筆記軟體** | 需要人親自整理 — 人變成瓶頸 |
| **知識型 Agent（L4）** | Agent 在夜間先把資料「消化」成結構化知識頁與知識圖，查詢時走索引與圖譜，知識會**複利成長** |

L4 不是反對 RAG，而是讓 RAG 成為「知識型 Agent」的其中一個檢索階段，不是全部。

## 3. 課程目標

完成 L4 課程後，學員應能：

1. 說明 Hermes Agent 與 L1 Chat、L2 Skill、L3 Workflow 的差異。
2. 說明第 2 節的七大設計原則，並用它們檢視自己的 Agent 設計。
3. 設計一個部門級 Hermes Agent 的使用情境、任務邊界、輸入來源與人工 Gate。
4. 建立 Hermes Agent 的知識作業結構：目的檔、領域結構檔、收件匣、任務佇列、觀察清單、任務追蹤檔、知識庫、索引快取。
5. 定義 L4 Agent 會吃什麼 input、執行什麼 process、產出什麼 output，以及用哪些 evidence 驗證。
6. 設計 ingest、query、update、lint、簡報、探勘、知識圖整合的運作流程。
7. 將 L2 Skill 與 L3 Workflow 接入 Hermes Agent 的任務鏈。
8. 建立 Agent 操作 Runbook、權限表、證據表、Stage Gate 與上線檢查表。
9. 判斷哪些任務適合 L4，哪些任務必須留在 L1-L3 或延後到 L5。

## 8. 技能組成：主控技能 + 專業技能

L4 Hermes Agent 的能力不是一個大 Prompt，而是「1 個主控技能 + 數個專業技能」的組合。課程教學員理解這個組合模式，而非背特定技能名稱。

| 技能類型 | 課程定位 | 學員要會什麼 |
|---|---|---|
| **主控技能（知識庫管理）** | Hermes Agent 的協調中樞 | bootstrap、ingest、query、update、lint、知識圖整合、schema 演進 — 主控技能負責「編排」，把工作分派給專業技能 |
| **來源分析技能** | 嚴謹來源分析 | 將 PDF、文章、報告、法規、影片逐一轉成結構化分析 |
| **關鍵字抽取技能** | 知識擴張 | 從新資料抽出關鍵字候選，交由人工 approve / reject（閉環的關鍵一環） |
| **自主探勘技能** | 自主發現 | 根據觀察清單到外部來源找 metadata，加入佇列，但不直接 ingest |
| **簡報生成技能** | 營運節奏 | 產生晨間簡報、晚間預覽、定期 ping、週度知識圖摘要 |

> **技能組合原則**：主控技能只負責「編排與決策」，專業技能只負責「單一專精工作」。新增能力時是「加一個專業技能」，不是「把主控技能改肥」。這讓 Agent 能力可擴充、可審查、可替換。

## 9. L4 Stage Gate 控制

| Gate | 檢查問題 | 必備 Evidence | 判定 |
|---|---|---|---|
| Gate 4A：環境可用 | Hermes、模型、gateway、權限是否可用？ | 環境健檢結果、model list、gateway status | Pass / Fail |
| Gate 4B：知識底座可用 | 知識庫、領域結構、目的、索引是否建立？ | 領域結構檔、目的檔、索引快取、初始化 log | Pass / Fail |
| Gate 4C：Ingest 閉環可用 | Agent 能否吃文件、分析、寫入、索引、留 Log、抽關鍵字？ | 來源頁、概念/實體/主張頁、log、索引查詢、關鍵字候選 | Pass / Fail |
| Gate 4D：查詢與更新可用 | Agent 能否用證據回答、回寫結果、保留紀錄？ | query record、update diff、引用來源、人工審核紀錄 | Pass / Fail |
| Gate 4E：營運與治理可用 | 是否有排程、簡報、觀察清單、佇列、人工 Gate、失敗模式清單？ | 排程設定、簡報、佇列、觀察清單、權限表、失敗模式清單、Gate 簽核 | Pass / Fail |

未通過 Gate 4A-4C 時，不進入部門上線。未通過 Gate 4D-4E 時，可以做展示，但不能宣稱為可營運的 L4 Agent。

## 11. 課堂練習

### 練習 1：L4 任務拆解

將「每週整理客訴與改善建議」改寫為 Hermes Agent 任務。

必填：

- 使用者是誰？
- Agent 哪些事白天做、哪些事夜間做？（落實原則 2.1）
- 需要吃哪些文件、系統與 Workflow？
- 哪些動作只能產生草稿，不能自動送出？
- 主管用什麼 evidence 驗收？

### 練習 2：建立目的與領域結構

為一個部門 Agent 建立：

- 目的檔（為什麼做這個 Agent — 相關性過濾依據）
- 領域結構檔（來源類型、概念類型、禁止事項、領域紅旗）

### 練習 3：Ingest 一份資料並走完閉環

輸入一份 PDF、SOP、FAQ 或報告，產出：

- 來源分析
- 來源頁
- 關鍵字候選（閉環關鍵一環，不可略過）
- log
- 索引查詢結果

### 練習 4：產生簡報

根據收件匣、任務佇列、觀察清單、任務追蹤檔產生一份主管可讀的晨間簡報。

### 練習 5：建立失敗模式清單

每組依原則 2.6，寫出 3 個自己部門可能遇到的 Agent 失敗模式，並為每個失敗模式定一條「不可重蹈」規則。

### 練習 6：設計 Gate 4

每組設計自己的 Gate 4A-4E 判定表，並說明哪些 evidence 不足時不能上線。

## 15. L4 到 L5 的銜接條件

只有當 L4 已具備以下能力時，才建議進入 L5 ClawTeam：

1. 至少 1 個 Hermes Agent 通過 Gate 4A-4E。
2. Agent 的 input、process、output、evidence 都可追溯。
3. 已有穩定的知識庫記憶、Log、索引與簡報節奏。
4. 七大設計原則皆已落實（特別是知識複利閉環與失敗模式清單）。
5. 已有至少 2 個可重複呼叫的 L2 Skill。
6. 已有至少 1 個穩定的 L3 Workflow。
7. 高風險決策有人工 Gate 與責任 Owner。

L5 不是把多個不穩定 Agent 放在一起，而是讓已經可治理的 L4 Agent 進入團隊協作。
