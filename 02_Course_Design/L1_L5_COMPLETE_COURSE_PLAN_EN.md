# Complete L1-L5 Course Plan and Classroom Content

Last updated: 2026-05-13

## 1. Course Design Goals

This course is not simply about teaching tools; it guides the enterprise along the L1-L5 maturity progression to build verifiable AI organizational capability step by step.

Every level must define:

- Course objectives.
- Target audience.
- Pre-class requirements.
- Classroom content.
- In-class hands-on practice.
- Post-class assignments.
- Completion criteria.
- Verifiable deliverables.
- The Stage Gate for advancing to the next level.

## 1.5 Pre-Class Story: Show the Future Before the Curriculum

The L1-L5 course should begin by helping clients imagine the capabilities they will have after training. Clients are not attracted to tools first; they are attracted to a believable future work scenario.

Recommended opening story:

> Three months later, employees no longer use disconnected AI tools on their own. Everyone logs in with their own OpenWebUI account, personal workspace, and permissions. Department heads convert expert methods into Skills. n8n connects Gmail, Sheets, Notion, CRM, APIs, and ERP into workflows. Hermes Agent produces daily briefings and follow-up lists. ClawTeam coordinates specialist agents for cross-functional tasks. The purpose of this course is to move the company toward that operating state, one level at a time.

| Level | Future Capability Clients Should Imagine | Evidence Produced in Class |
| --- | --- | --- |
| L1 | Every employee has a controlled AI workspace and knows the data boundaries | OpenWebUI account / group / permission sheet, personal workspace screenshots, Prompt Library |
| L2 | Departmental expert methods become reusable Skills | Skill Library, Skill template, non-author test records |
| L3 | AI performs work across systems instead of only answering questions | n8n Workflow PoC, Execution Log, integration sheet, human gate |
| L4 | Agents read data, query Wiki memory, call tools, produce briefings, and keep evidence | Hermes Agent task card, Wiki records, briefing, Gates 4A-4E |
| L5 | Multiple agents collaborate on cross-functional work | Agent Team role cards, task dispatch records, integrated report template |

The goal of this story is to make clients understand that they are not buying "a training course"; they are buying a capability path from personal AI use to an enterprise AI Operating Model.

## 3. Course Packaging Recommendations

### 3.1 Half-day Experience Course: 3 hours

Suitable for: first-time customer contact, executive alignment sessions, business development.  
Content: L1-L5 overview, 10-question rapid diagnostic, one scenario demonstration.  
Outputs: preliminary maturity assessment, course ratio recommendation, next-step proposal.

### 3.2 One-day Workshop: 6 hours

Suitable for: department managers, HR, seed personnel.  
Content: L1 + L2 + L3 scenario inventory.  
Outputs: draft AI usage policy, Prompt Library, Skill candidate list, Workflow candidate list.

### 3.3 Two-day Onboarding Bootcamp: 12 hours

Suitable for: seed team, IT, process owners.  
Content: L2 Skill construction + L3 n8n PoC + L4 Agent task card.  
Outputs: 3-5 Skills, 1 n8n PoC, 1 Hermes Agent task card.

### 3.4 Consulting Diagnostic Project: 2-4 weeks

Suitable for: leadership, cross-functional teams.  
Content: questionnaire, interviews, As-Is, gap analysis, To-Be, roadmap.  
Outputs: AI transformation diagnostic report, ROI tracking matrix, follow-up SOW.

## 4. L1 Chat AI: OpenWebUI

### 4.1 Course Objectives

Enable employees to safely use AI for daily knowledge work via a controlled AI gateway, and to establish enterprise AI usage boundaries. The focus of the enterprise edition of L1: every employee logs in to OpenWebUI with their own account and owns their own chat area; Admin / IT can manage accounts, roles, groups, models, features, and data permissions.

The complete design of the OpenWebUI enterprise enablement course is documented in `L1_OPENWEBUI_COURSE_PLAN.md`.

### 4.2 Target Audience

- All employees.
- Departmental seed personnel.
- HR / training functions.
- IT and information security contacts.

### 4.3 Pre-class Requirements

| Item | Description |
| --- | --- |
| AI gateway | OpenWebUI or a company-approved AI tool |
| Accounts | Each learner has their own test account; no shared accounts |
| Admin account | IT / HR / AI administrator has Admin privileges |
| Groups / permissions | Pre-plan department groups, model visibility, file upload, Web Search, and sharing policy |
| Sample data | Low-sensitivity documents, meeting minutes, email samples |
| Draft policy | If none exists, create the first version during class |

### 4.4 Classroom Content

| Module | Duration | Content | Method |
| --- | ---: | --- | --- |
| L1-L5 maturity introduction | 30 min | The evolution from personal AI use to Agent Team | Lecture |
| OpenWebUI personal operations | 45 min | Login, personal chat area, model selection, conversation, files, history | Demo |
| OpenWebUI Admin management | 60 min | Account approval, roles, groups, permissions, model and feature governance | Demo + hands-on |
| Prompt fundamentals | 60 min | Role, task, background, constraints, output format | Lecture + practice |
| Daily-work hands-on | 90 min | Email, summaries, meeting minutes, executive reports | Hands-on |
| AI usage policy | 45 min | Can input / must not input / requires human confirmation | Workshop |
| Outcome review | 30 min | Showcase Prompts and outputs | Group feedback |

### 4.5 In-class Hands-on Practice

- Write a customer email draft.
- Each learner creates 2 chat topics using their own account.
- Admin creates or approves test accounts and configures at least 1 department group.
- Convert meeting minutes into action items.
- Distill a long document into an executive summary.
- Build 3 personal frequently-used Prompts.
- Judge 10 data cases for whether they may be input into AI.

### 4.6 Post-class Assignments

- Each learner submits 3 work Prompts.
- Each department submits a list of 5 high-frequency tasks.
- IT / Admin submits the account, group, permission, and model-visibility configuration sheet.
- IT / HR completes the draft AI usage policy.

### 4.7 Completion Criteria

| Criterion | Verification Method |
| --- | --- |
| Learner can use OpenWebUI to complete basic tasks | In-class hands-on outputs |
| Each learner can log in with their own account and create personal chats | Login screenshot, personal chat screenshot |
| Admin can manage accounts, roles, groups, models, and permissions | Admin Panel screenshot, permission sheet |
| Learners understand data input boundaries | Data-case judgment sheet |
| Department has an initial Prompt Library | Prompt list |
| Company has a draft AI usage policy | Policy document |

### 4.8 L1 Deliverables

- Draft AI usage policy.
- First version of the Prompt Library.
- OpenWebUI usage workflow.
- OpenWebUI Admin Runbook.
- Account / group / permission / model-visibility configuration sheet.
- High-frequency task list.
- L2 Skill candidate list.

### 4.9 Gate 1: Eligibility to Enter L2

Pass criteria:

- A controlled gateway or approved tool exists.
- Each learner has their own account and personal chat area.
- Admin can approve accounts and configure roles, groups, models, and feature permissions.
- Seed personnel have completed the L1 hands-on practice.
- A policy on permitted / prohibited data exists.
- At least 10 high-frequency departmental tasks have been catalogued.

## 7. L4 Auto Agentic AI: Hermes Agent

### 7.1 Course Objectives

Design a controlled, verifiable, and operable Hermes Agent that uses a Wiki as long-term memory, invokes L2 Skills and L3 Workflows, completes data ingestion, analysis, query, update, reminders, and reporting, and preserves human Gates.

The complete L4 course design is documented in `L4_HERMES_AGENT_COURSE_PLAN.md`. This section serves as the L1-L5 syllabus summary.

### 7.2 Target Audience

- Department managers.
- Process Owners.
- IT.
- Seed personnel.
- Knowledge workers.
- Executive assistants / PMs.

### 7.3 Pre-class Requirements

| Item | Description |
| --- | --- |
| L2 Skills | Skills the Agent can call |
| L3 Workflows | Workflows the Agent can start |
| Task scenarios | Weekly reports, customer service, operations, sales, administration, research, document management |

| Technical foundation | Hermes Agent, the five core skills, tools, SQLite / FTS index, scheduling mechanism |
| Permission sheet | What the Agent can and cannot do |
| Human Gate | When human confirmation is required |

### 7.4 Agent Task Card Format
