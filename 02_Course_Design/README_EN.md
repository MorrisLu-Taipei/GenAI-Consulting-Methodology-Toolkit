# 02 Course Design

This directory holds the course design rationale and the course modules.

- `COURSE_REQUIREMENTS_AND_COMPANY_PROFILE.md`: L1-L5 course requirements checklist, company profile survey, and deployment considerations for cloud AI / hybrid / fully on-premises.
- `COURSE_MODULE_MATRIX.md`: L1-L5 course outlines, hands-on exercises, post-class deliverables, and course packaging.
- `L1_L5_COMPLETE_COURSE_PLAN.md`: The complete L1-L5 delivery plan, covering course objectives, classroom content, hands-on practice, post-class assignments, completion criteria, and Stage Gates.
- `L1_OPENWEBUI_COURSE_PLAN.md`: The L1 enterprise enablement course referencing the OpenWebUI playlist, covering per-user login, personal chat area, Admin Panel, accounts/roles/groups/permissions, model governance, data policy, and a video reference map.
- `L2_ANTIGRAVITY_COURSE_PLAN.md`: The L2 engineering training referencing the three Google Antigravity codelabs, covering Agentic IDE, App Prototype, Unit Test, GCP Serverless Pipeline, and Gate 2A-2E.
- `L3_N8N_TIGERAI_COURSE_PLAN.md`: The L3 deep-dive course referencing the n8n / OpenGenie videos on the TigerAI channel, covering Gemini, multimodal, Sub-workflow, Data Tables, Webhook, customer service / HR / marketing PoCs, GitHub backup, and Gate 3A-3G.
- `L4_HERMES_AGENT_COURSE_PLAN.md`: The L4 deep-dive course referencing the Hermes `starter-kit-v2`, covering Wiki memory, the five core skills, ingest/query/update, briefing/discovery, evidence, and Gate 4A-4E.
- `L5_CLAWTEAM_COURSE_PLAN.md`: The L5 deep-dive course using [HKUDS/ClawTeam](https://github.com/HKUDS/ClawTeam) (MIT License) as the implementation platform, covering the five-layer architecture (Team / Workspace / Task / Inbox / Transport), git worktree isolation, CLI hands-on (`team spawn-team`, `task --blocked-by`, `inbox send/broadcast`, `board show/live`), three localized scenarios (manufacturing / hospital / retail), and Gate 5. Full citation terms: `../90_References/CLAWTEAM_REFERENCE.md`.
